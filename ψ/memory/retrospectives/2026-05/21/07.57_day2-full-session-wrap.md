## Honest Feedback

**Friction 1: Browser verification debt is now a liability.** sidebar.js is a JS-dependent feature. If it has a rendering bug, every article shows a blank sidebar on GitHub Pages. Code review said "ship-ready" but code review is not a functional test. 31 articles, zero visual verification, ever. This needs a gate, not just a note.

**Friction 2: Research breadth over depth.** 8 articles this session, all "emerging" maturity. None have application evidence (lab experiments). The articles are reference material, not learnings in the DEVBOY sense. P'Nat's hidden curriculum is about execution discipline, not research volume. One article with a working lab experiment is worth more than three without.

**Friction 3: Agent efficiency varies widely.** 30 agents this session. Research agents (~2 min each) found real sources. Analysis agents (~1.5 min each) produced actionable findings. Verification agents (~35s each) confirmed correctness. But some research agents produced overlapping content. Better prompt engineering (more specific questions, smaller scope) would produce more value per agent.

## Lessons Learned

1. **Infrastructure fixes compound — build them first, not last.** sidebar.js: 15 min fix, saves 20 min/session. 5 sessions of deferral = ~100 min wasted on perl batch syncs that didn't need to happen.
2. **Compaction resilience gate is validated** — synthesis stubs in ψ/active/ survived 4 compaction boundaries. This is DEVBOY's first operationally proven gate.
3. **maw tile is the implementation bridge** — Extends article 003's conceptual coverage into concrete implementation. Friction 0.90 confirms: strong foundation, real delta.
4. **Gate hooks need enforcement, not just documentation** — sidebar gate_hook fired 3 times without action. A gate that doesn't block isn't a gate.
5. **Inline JS data > separate JSON for static sites** — No fetch, no CORS, no async. Works with file://. The pragmatic choice.
