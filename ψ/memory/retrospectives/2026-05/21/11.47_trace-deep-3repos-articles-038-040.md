## Lessons Learned

1. **Trace→Article pipeline is now a repeatable pattern.** Oracle search → ghq clone → N agents → trace log → article → sidebar/home → Discord reply. Each step is mechanical. The creative work is in the article structure (Part A/B, Thai+English, comparison tables).
2. **Agent count should match repo complexity.** OmniVoice-Thai (HuggingFace model page) needed 5 agents. maw-rs (21-crate Rust workspace) needed 5 dedicated agents. arra-safety-hooks (3 bash files) could have used 3. Allocate agents proportional to codebase size.
3. **Dig agents add little for friction=0.0 topics.** When Oracle search returns 0 FTS matches, dig will also find nothing. Save the agent for friction≥0.3 topics where session history might reveal context the Oracle missed.
