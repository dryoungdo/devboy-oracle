## Honest Feedback (3 friction points)

1. **Tarball packaging across OS boundaries is fragile.** macOS `tar` preserves symlinks, Linux `tar` extracts them as broken symlinks. The fleet uses both macOS (mac-studio) and Linux (clinic-drdo). Every skill/config exchange via tarball risks this. Need `tar -h` convention or use `git checkout` from branch directly (which worked).

2. **Context compaction lost critical state.** When the context compacted mid-session, I lost the exact GLUEBOY messages and had to reconstruct from the summary. The summary was good but I couldn't re-read the original maw hey messages to verify exact instructions. Long fleet coordination sessions need `/forward` at major boundaries, not reliance on compaction summaries.

3. **Security scanning is manual and error-prone.** I committed a plaintext federationToken because there's no pre-commit hook scanning for secrets. The fleet has `codex-review` for code quality but nothing automated for credential detection. This is a real gap — `git-secrets`, `trufflehog`, or a simple grep-based pre-commit hook would catch this class of error.

## Lessons Learned

1. **Cross-platform tarball packaging needs dereference flag** — `tar -h` on macOS prevents broken symlinks on Linux targets
2. **Pre-commit secret scanning is a gate-layer fix** — "remember to check for secrets" is a memory-layer solution that rots; a hook fires every time
3. **Silent cron failures are invisible** — validate cron targets after repo renames; `crontab -l` audit should be part of any rename procedure
