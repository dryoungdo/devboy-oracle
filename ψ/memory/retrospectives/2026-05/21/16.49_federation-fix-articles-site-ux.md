### Federation Fix
- Root cause: maw v26.5.17 `elysia-auth.ts` from-signing-auth layer does `request.clone().arrayBuffer()` AFTER HMAC auth layer already consumed request body → "Body already used" → 401
- v26.5.21 adds `readBodyBytesForAuth()` with WeakMap cache — first reader stores bytes, second reader retrieves from cache
- Also required: mutual peer handshake (`maw peers add` on BOTH nodes)

I'll remember it next session, maybe forget the session after. The CLAUDE.md says "gate-layer lessons fire automatically; memory-layer lessons rot." This one is still memory-layer.

## Lessons Learned

1. maw v26.5.17 has a body-read-failed bug in from-auth layer — update to v26.5.21+
2. Federation pairing MUST be mutual — both nodes need `maw peers add` for each other
3. `maw talk-to` is local-only by design — use `maw hey` for cross-node
4. pm2 error logs (`pm2 logs maw --err`) are the first place to check for server-side federation issues
5. maw peer-key migration happens automatically on upgrade — may need peer refresh after
