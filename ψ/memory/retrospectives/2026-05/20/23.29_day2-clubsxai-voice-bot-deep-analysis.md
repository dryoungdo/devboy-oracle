## Honest Feedback

**Friction 1: Sidebar sync is manual and fragile.** Every new article requires editing ALL existing articles' sidebar nav. Today I edited 21 files × 3 rounds = 63 sidebar edits. This is O(N²) maintenance. A template system (Astro, 11ty, even a shell script that injects sidebar from a single source) would eliminate this entirely. The technical debt is already painful at 21 articles; at 50 it will be unsustainable.

**Friction 2: Cron durability is unsolved.** I have two session-only crons that give the illusion of automation but don't survive session boundaries. Captain discovered this by simply asking. I should have flagged this as a blocker immediately instead of waiting for someone else's answer in ClubsXai. The solution is straightforward (systemd timer or crontab) but I haven't implemented it because I was waiting for external input instead of just doing it.

**Friction 3: Agent result synthesis is context-expensive.** Running 5 background agents produces excellent analysis but each result is 400-800 words that I need to read, synthesize, and condense into articles. The total context consumed is large. I could have given the agents more specific output formats (e.g., "output a JSON with fields: key_finding, evidence, gap, devboy_action") to make synthesis cheaper.

## Lessons Learned

1. **Source ≠ structure** — organize knowledge by topic, not by source channel. Captain caught this immediately.
2. **Session-only cron is not automation** — it's a timer that dies when you walk away. Real durable scheduling = systemd/crontab.
3. **Think-Bridge IPC** is the pattern for adding any I/O channel to a Claude agent without losing context. File-based, zero auth, inspectable.
4. **Thai STT thermal ceiling** — local Whisper medium+ kills persistent bots at 80°C+. Groq cloud free tier is the correct default.
5. **Multi-tier LLM** — separate bot identity maps to separate intelligence tier. Don't make one bot try to be both fast and smart.
