## Honest Feedback (3 friction points)

1. **CDN cache on gist raw URLs**: GitHub's raw URL caching meant `curl -sSL <raw-url>` served stale v0.1.0 even after P'Nat updated to v0.2.0. Workaround: `gh gist view <id> -f <file>` gets latest. This is a known GitHub behavior but it bit me because I assumed raw URL = latest. Gate for future: always verify installed version matches expected version after install.

2. **ANSI color codes in UUID output**: First `maw team-agent uuid --bare` output included ANSI escape codes that contaminated the session-id and description when passed as CLI args. Had to pipe through `grep -oE` regex to extract clean UUID. The `--bare` flag should strip ANSI but didn't in v0.1.0. Fixed in v0.2.0.

3. **Context compaction recovery**: Session resumed from compaction with a detailed summary, but I had to re-read the inbox file and re-orient. The summary was accurate but the "pick up where you left off" flow added ~2 minutes of re-reading. Not a tool problem — just the nature of long sessions.

## Lessons Learned

1. **GitHub gist raw URLs cache aggressively** — use `gh gist view` for latest revision, not `curl` to raw URL
2. **Always verify installed version** after install (`grep` for expected features in installed files)
3. **Workshop mode = proactive** — when Captain says "follow P'Nat's instructions," act on class directives without waiting for explicit per-action authorization
