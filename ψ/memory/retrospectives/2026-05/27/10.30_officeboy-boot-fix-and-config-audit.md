## Honest Feedback

**Friction 1: Can't test plugin resolution from inside a running session.** The Claude plugin loader is opaque from within a running Claude instance. No `claude plugin resolve @name` command exists. I had to commit, push, boot, observe, diagnose, fix, re-commit, re-boot. Two round trips for something that could have been a single test. This is a structural friction — either the plugin system needs introspection tools, or I need a pre-boot validation script.

**Friction 2: Gate_hook false positive undermines trust in health checks.** The `discord_mcp_connected()` function uses `claude mcp list` which is global across all sessions on the machine. When DEVBOY's Discord MCP is healthy, the check passes even if OFFICEBOY's MCP failed to start. The per-PID `listener_has_discord_mcp_child` check should have caught this, but on first boot (custom registry failure) it somehow passed. Need to investigate whether `pstree` was matching DEVBOY's bun child through tmux ancestry. Both DEVBOY and OFFICEBOY start.sh have this same blind spot.

**Friction 3: OFFICEBOY Phase 0 shipped content without infrastructure.** I focused on the audit report, verify-fresh checklist, and 10-article roadmap — good content. But I didn't set up oracle-build, proper settings.json, AGENTS.md, or .gitignore. OFFICEBOY was born with a brain (CLAUDE.md content) but no skeleton (oracle-build + shared doctrine + operational scripts). Captain noticed immediately when he read the CLAUDE.md — "it's not feels right." The missing 636 lines of fleet doctrine IS what doesn't feel right.

## Lessons Learned

1. **Claude Code plugin `@` notation only accepts known/installed registries** — custom directory names under `~/.claude/plugins/cache/` are rejected. Per-oracle isolation requires either patching the shared cache or an upstream plugin feature.
2. **`claude mcp list` is global** — health checks using it can false-positive when another oracle on the same machine has a healthy MCP. Per-session health verification needs per-PID process tree checks ONLY, not global MCP list.
3. **Oracle birth requires infrastructure before content** — oracle-build system (identity + shared doctrine generator), proper settings.json (permissions + hooks), AGENTS.md, .gitignore. Without these, the oracle has identity but no operational foundation.
