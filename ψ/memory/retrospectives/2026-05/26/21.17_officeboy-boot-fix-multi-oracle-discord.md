## Lessons Learned

1. **Claude Code `settings.json` does NOT support `mcpServers`** — MCP servers are defined via `--channels` (plugin system) or `.mcp.json` files, not in settings.json
2. **`--channels plugin:X@Y` — the `@Y` part is a registry name**, resolved to `~/.claude/plugins/cache/Y/`. Full paths don't work. Custom registry names (creating a new directory under the cache) may work but is unverified.
3. **Multi-oracle Discord on same machine**: each oracle needs its own `.mcp.json` with its own `DISCORD_STATE_DIR`. The proven pattern is patching at startup (last to start wins). The better pattern is per-oracle registry directories.
