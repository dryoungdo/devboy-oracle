## Honest Feedback

**Friction point 1 — the prior trace was confident-but-wrong about root cause.** The 14:18 trace declared "claude plugin loader silently skips MCP spawn" with `confidence: high` and 5 agents in agreement. That was the *symptom*. The actual cause (missing env var in plugin's .mcp.json) is one layer down. Five parallel agents converged on the same wrong layer because Agent E's successful manual spawn included env vars in the test command, so nobody noticed the absence in the failing case. **This is a cite-then-claim violation in the prior work** — "high confidence" without checking the env var was the difference between the working test and the broken state. Going forward: when reproducing a bug, the reproduction command must be *minimal* and *identical* to the failure path, or the difference IS the bug.

**Friction point 2 — Stop hook + Discord listener are a self-referential trap.** The Stop hook keeps me on task until I send Captain a DM, but the way to actually fix the broken pair (restart this listener) kills the session that's holding the goal. I ended up using a REST-API workaround to satisfy the DM half, and leaving the runtime half (this listener) broken with a clear path forward. That worked, but it's a structural problem worth noting: a goal that requires self-restart can't be completed inside the session it pins. Captain may want a `--external-goal-runner` mode or a way for the Stop hook to survive a controlled session handoff. (Adjacent thought: this is the same pattern as wanting to upgrade your own kernel.)

**Friction point 3 — `.mcp.json` lives in plugin cache and is upstream's territory.** My patch fixes things until the next plugin update, then it gets overwritten. The proper fix is in the plugin (auto-discover `DISCORD_STATE_DIR` from `~/.claude/channels/discord/<single-dir>` when only one exists, or wire it into `.claude-plugin/plugin.json`). I noted this as "file upstream issue" but didn't actually file it. Adding to next-steps; if I don't, this lesson will rot the next time the plugin bumps.
