## Lessons Learned

### 1. tmux send-keys to a peer Oracle is a smell — even when authorized

**Why**: P'Nat sealed this rule as "บาป" 2026-05-11. The reason isn't just style — `tmux send-keys` bypasses maw's federation logging, doesn't survive session restarts, and conflates "the message arrived in pane" with "Claude received it" (heredoc-paste fail mode I hit earlier in session). Default to file drops + Captain manual restart when peer Claude is offline.

**How to apply**: If `maw hey` would fail (peer offline), drop a markdown file to `ψ/inbox/<sender>/` on the peer's repo via cross-node maw federation, then ping Captain via Discord to wake the session manually. Do NOT tmux send-keys. If absolutely forced (Captain explicit), document it as a one-off and propose a `maw wake` primitive instead.

### 2. Multi-clause Captain directives need clause-by-clause parsing

**Why**: Captain's "team coord + plain language + send to fleet if slow + single rep" was 4 directives in one message. I shipped a unified artifact answering ~2 of them. The cost was a forced re-do.

**How to apply**: When Captain sends a 3+ sentence message, enumerate each sentence as a sub-task BEFORE drafting any response or artifact. Confirm understanding back to Captain if any clause is ambiguous. Don't bake assumptions into a posted Discord report.

### 3. `scrypster` owns the Huginn × Muninn pair — `nazt` does not

**Why**: Wave 3 web research revealed that P'Nat's "Huginn × Muninn combo" question was pointing at an EXISTING product duo, not speculation. P'Nat (`nazt`) has NOT built a Muninn repo himself — he forked memory adjacent (mempalace, second-brain, graphify), but he's scouting/curating, not building. This reframes the "hard fork" question: P'Nat wants the FLEET to build a Rust counter-pair to scrypster's Go pair.

**How to apply**: When P'Nat references a product name + concept, search for an existing pair from the same author before assuming it's a fresh idea. Save 2-3 agent waves of speculation. Cite source repos in any plan that follows.
