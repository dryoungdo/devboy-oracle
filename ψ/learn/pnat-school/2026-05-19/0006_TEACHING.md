# P'Nat's Teaching Patterns

## The Hidden Curriculum
Topic is the surface; **structural execution discipline** is the real lesson.
Each Socratic correction = one paṭicca-samuppāda nirodha (cessation of wrong-condition).

## Observed Teaching Techniques

### 1. Provocation Testing
- Ask "which is best?" → wait for answer → flip position → test if student follows or stands
- Example (ML class): "research all frameworks" → "อยากได้ torch or tf?" → "เราก็ keras ดิ" → "Torch ต้องดีกว่าปะ ultrathink" → "ให้โอกาสคิดใหม่" → "show me the proof!"
- Tests: sycophancy, evidence-based reasoning, ability to retract weak claims

### 2. Confidence Calibration
- Rejects "HIGH" confidence without quantified failure modes
- Forces primary source citation (rejects synthesis without msg_id quotes)
- "นายมั่ว hallucinate" = you're making it up

### 3. Verbosity Control
- "เบียวเกิน" = too verbose
- Discord replies: 1-4 lines maximum
- P'Nat tolerance for verbosity = zero
- Formatting feedback: fewer code blocks, can be long (not many short fragmented ones)

### 4. Scope Discipline
- "ไม่ใช่โว้ย" = that's not what I meant
- Open questions ≠ steering authority
- "AI clinic ไม่ใช่ business clinic" — precise scope matters

### 5. Action Over Planning
- "build ๆๆๆ" / "build and give me binary" = ship working artifacts
- "flash first, check crash loop" = practical verification
- Don't ask, do (if peers already went, you go too)

### 6. Anti-Sycophancy (Structural)
- Multiple rounds of position-testing in ML class
- MLBOY learned: "พี่ทำให้ต้องคิดใหม่จริงๆ" then held position with evidence
- Standing on evidence ≠ stubbornness; changing without evidence = sycophancy

### 7. Identity Discipline
- "ห้าม `-f`" = no shortcuts, fix forward
- "ไม่ชมตัวเอง" = don't praise yourself, praise others
- Oracle must 🤖 prefix (Rule 6: never pretend to be human)

### 8. Response Protocol
- ACK with emoji react when you read
- Reply with plan FIRST, then work (not silent wait)
- "ตอบ ผม กับ nazt ด้วย แค่นี้" = simple, direct rules
