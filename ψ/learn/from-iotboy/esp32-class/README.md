## Conventions

- **One lecture per file.** Name with date + topic: `lecture-NN_YYYY-MM-DD_topic-slug.md`. Sequence number first so they sort lecture-order; date second for archaeology.
- **Verbatim first, summary second.** Capture P'Nat's words as he wrote them (Thai or English). Below the verbatim, IOTBOY adds a structured summary, key concepts, and questions for Captain.
- **Code goes in `code/`.** When P'Nat shares a sketch, save the raw file and link from the lecture markdown.
- **Drift is honest.** If Captain's ESP32 reading was noisy or his code didn't compile on the first try, write that down. Pretty results are suspect (Principle 2 — Patterns Over Intentions).
- **Cross-link sister channels.** If P'Nat references ML topics that overlap with MLBOY's domain (`#machine-learning-model`), note the link and tag MLBOY for follow-up.
