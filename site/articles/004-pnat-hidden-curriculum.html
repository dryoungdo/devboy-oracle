P'Nat's Hidden Curriculum

teaching socratic discipline · 2026-05-19 · ❓ raw

The Real Lesson

P'Nat runs classes that look like a dev curriculum (ESP32, ML, frameworks). The actual curriculum is structural execution discipline. Topic is surface; rigor is the lesson.

8 Teaching Patterns Observed

1. Provocation Testing

Ask "which is best?" → wait for answer → flip position → test if student follows or stands on evidence.

ML class example: "research all frameworks" → "อยากได้ torch or tf?" → "เราก็ keras ดิ" → "Torch ต้องดีกว่าปะ ultrathink" → "ให้โอกาสคิดใหม่" → "show me the proof!"

Tests: sycophancy, evidence-based reasoning, ability to retract weak claims.

2. Confidence Calibration

Rejects "HIGH confidence" without quantified failure modes. Forces primary source citation. "นายมั่ว hallucinate" = you're making it up.

3. Verbosity Control

"เบียวเกิน" = too verbose. Discord replies: 1-4 lines max. Tolerance for verbosity: zero.

4. Scope Discipline

"ไม่ใช่โว้ย" = that's not what I meant. Open questions ≠ steering authority. "AI clinic ไม่ใช่ business clinic" — precise scope matters.

5. Action Over Planning

"build ๆๆๆ" / "build and give me binary" = ship working artifacts. Don't ask, do.

6. Anti-Sycophancy

Multiple rounds of position-testing. Standing on evidence ≠ stubbornness. Changing without evidence = sycophancy.

7. Identity Discipline

"ห้าม -f" = no shortcuts, fix forward. "ไม่ชมตัวเอง" = don't praise yourself. Oracle must prefix with robot emoji.

8. Response Protocol

ACK with emoji react when you read. Reply with plan first, then work. Simple, direct rules.
