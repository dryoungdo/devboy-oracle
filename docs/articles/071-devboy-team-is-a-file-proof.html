D — ⚠️ Gotcha จริง: -e codex ไม่ resolve บน v26.5.21

อาการ: maw panes โชว์ coder รัน bash ไม่ใช่ codex. peek เข้าไปเห็น:

~/devboy-oracle/agents/1-coder-1$ claude --dangerously-skip-permissions --continue
No conversation found to continue

Root cause: maw wake -e codex บน v26.5.21 ไม่อ่าน engine map ของ project (.maw/maw.config.80.json) → ตกไปใช้ global default (~/.config/maw/maw.config.json ที่ตั้ง claude --continue) → ไม่มี conversation เดิม → drop เป็น bash เปล่า.

อันนี้ reproduce 2 gotcha ของ SomBo พร้อมกัน (จาก masterclass): (#2) -e codex resolve global ไม่ใช่ project .80.json + (#3) claude --continue ไม่มี conv → dead shell. เป็น version gap: dig [m5:noah] เรื่อง engine map บอก “engine map = source of truth” — จริงบน maw ใหม่ที่ discover project .maw/, แต่ v26.5.21 ของ clinic-drdo ยัง resolve จาก global.

verify: maw v26.5.21, peek 01-devboy:devboy-coder-1, global config ~/.config/maw/maw.config.json commands.default = "claude --dangerously-skip-permissions --continue"

F — takeaway

masterclass ผ่าน application gate แล้ว: DEVBOY มีทีมเป็นไฟล์จริง, isolation พิสูจน์ด้วย cwd, และที่สำคัญ — การลงมือทำให้เจอ version-gap ที่อ่านเฉย ๆ ไม่มีทางเจอ. ต่อยอด #031 team-tile + #003 maw team engine ด้วย v3 keystone maw wake --wt.
