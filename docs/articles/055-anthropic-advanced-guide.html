A — Claude Agent SDK

เปลี่ยน Claude Code จาก CLI เป็น library — สร้าง AI agent ที่อ่านไฟล์, รัน command, edit code ได้เอง ผ่าน Python หรือ TypeScript

A.1 — Install

# Python
pip install claude-agent-sdk

# TypeScript (bundles native Claude Code binary)
npm install @anthropic-ai/claude-agent-sdk

A.2 — Hello World

# Python
import asyncio
from claude_agent_sdk import query, ClaudeAgentOptions

async def main():
    async for message in query(
        prompt="Find and fix the bug in auth.py",
        options=ClaudeAgentOptions(allowed_tools=["Read", "Edit", "Bash"]),
    ):
        print(message)

asyncio.run(main())
// TypeScript
import { query } from "@anthropic-ai/claude-agent-sdk";

for await (const message of query({
  prompt: "Find and fix the bug in auth.ts",
  options: { allowedTools: ["Read", "Edit", "Bash"] }
})) {
  console.log(message);
}

A.4 — Subagents ใน SDK

# Python — define custom subagent
async for message in query(
    prompt="Use the code-reviewer agent to review this codebase",
    options=ClaudeAgentOptions(
        allowed_tools=["Read", "Glob", "Grep", "Agent"],
        agents={
            "code-reviewer": AgentDefinition(
                description="Expert code reviewer.",

                tools=["Read", "Glob", "Grep"],
            )
        },
    ),

A.5 — Hooks (Programmatic)

SDK hooks ใช้ callback functions — ไม่ต้อง shell command เหมือน CLI

# Python — log all file changes to audit
async def log_file_change(input_data, tool_use_id, context):
    file_path = input_data.get("tool_input", {}).get("file_path", "unknown")
    with open("./audit.log", "a") as f:
        f.write(f"{datetime.now()}: modified {file_path}\n")
    return {}

options=ClaudeAgentOptions(
    permission_mode="acceptEdits",
    hooks={
        "PostToolUse": [
            HookMatcher(matcher="Edit|Write", hooks=[log_file_change])
        ]
    },
)

Hook types: PreToolUse, PostToolUse, Stop, SessionStart, SessionEnd, UserPromptSubmit

A.7 — Sessions (Resume + Fork)

# Capture session ID from first query
async for message in query(prompt="Read the auth module", ...):
    if isinstance(message, SystemMessage) and message.subtype == "init":
        session_id = message.data["session_id"]

# Resume with full context
async for message in query(
    prompt="Now find all places that call it",
    options=ClaudeAgentOptions(resume=session_id),
):
    print(message)

C — Prompt Caching

ลด cost ได้ 90% สำหรับ cached content — cache read = 10% ของ base input price

C.1 — Automatic Caching (ง่ายสุด)

response = client.messages.create(
    model="claude-opus-4-7",
    max_tokens=1024,
    cache_control={"type": "ephemeral"},  # Top-level = auto
    system="You are an AI assistant...",
    messages=[{"role": "user", "content": "Analyze this..."}],
)

System auto-applies cache breakpoint ที่ block สุดท้ายที่ cache ได้ — เลื่อนตามใน multi-turn

# Load cache BEFORE users arrive
prewarm = client.messages.create(
    model="claude-opus-4-7",
    max_tokens=0,            # Key: zero output
    system=[{
        "type": "text",
        "text": "You are an expert...",

    }],

C.6 — Common Mistake

# ❌ WRONG — breakpoint on changing content
system=[
    {"type": "text", "text": "Long instructions"},
    {"type": "text", "text": "Timestamp: 2024-01-15 10:30:45",
     "cache_control": {"type": "ephemeral"}}  # Hash changes!
]

# ✅ CORRECT — breakpoint on stable content
system=[
    {"type": "text", "text": "Long instructions",
     "cache_control": {"type": "ephemeral"}},  # Stable!
    {"type": "text", "text": "Timestamp: 2024-01-15 10:30:45"}
]

D — Extended Thinking

Claude แสดง step-by-step reasoning ก่อนตอบ — ได้คำตอบดีขึ้นสำหรับปัญหาซับซ้อน

D.1 — Basic Usage

response = client.messages.create(
    model="claude-sonnet-4-6",
    max_tokens=16000,
    thinking={"type": "enabled", "budget_tokens": 10000},
    messages=[{"role": "user", "content": "Prove there are infinite primes n mod 4 == 3"}],
)

for block in response.content:
    if block.type == "thinking":
        print(f"Thinking: {block.thinking}")
    elif block.type == "text":
        print(f"Answer: {block.text}")

D.2 — Display Modes

Modeได้อะไรDefault On
"summarized"Summary of reasoningOpus 4.6, Sonnet 4.6
"omitted"Empty thinking + signature onlyOpus 4.7

ทั้งสอง mode เสียค่า full thinking tokens เท่ากัน — omitted แค่ไม่ส่ง content กลับ (เร็วกว่า)

D.3 — Adaptive Thinking (Opus 4.7)

Opus 4.7 ใช้ thinking: {"type": "adaptive"} แทน manual — Claude จัดการ budget เอง

D.4 — Interleaved Thinking

# Without: [thinking] → [tool] → [tool] → [text]
# With:    [thinking] → [tool] → [thinking] → [tool] → [thinking] → [text]

Claude คิดระหว่าง tool calls — ได้ reasoning ที่ดีกว่าสำหรับ multi-step tool use

D.5 — ข้อจำกัด

  • budget_tokens ต้องน้อยกว่า max_tokens (ยกเว้น interleaved)
  • Tool choice "any" หรือ {"type": "tool"} ใช้ไม่ได้กับ thinking
  • ต้องส่ง thinking blocks กลับใน multi-turn tool use
  • ใช้กับ max_tokens: 0 (pre-warming) ไม่ได้
