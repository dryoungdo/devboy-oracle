Fleet v3 Relevance

สิ่งที่ Fleet v3 ใช้อยู่แล้ว

  • Claude Code — primary agent runtime สำหรับทุก BOY
  • maw-js — orchestrator + plugin system (Discord, arra-oracle)
  • Oracle Ecosystem — arra-oracle MCP + ψ/ vault + /rrr + /trace + /learn = knowledge backbone ของ fleet ทั้งหมด
  • thClaws — installed บน Samsung Note 20 Ultra สำหรับ mobile agent access

สิ่งที่ควร adopt จาก Hermes

  • Self-improving skills — maw plugin system ทำได้ถ้า plugin ถูก generate จาก session data
  • Trajectory compression — DEVBOY sessions = training data สำหรับ specialized models
  • MCP server mode — Oracle expose ตัวเองให้ Oracle อื่นเรียก — natural inter-oracle comm

สิ่งที่ควร adopt จาก thClaws

  • 3 cache breakpoints — 46–74% savings = ลด API cost อย่างมีนัยสำคัญ
  • autoLearn — automatic KMS ingestion = /rrr ที่ fire เอง ไม่ต้องรอ end-of-session
  • AI Agent API — /agent/run endpoint = GLUEBOY สั่ง BOY ผ่าน HTTP ได้
  • AGENTS.md — vendor-neutral standard ที่ work กับ CLAUDE.md ด้วย

DEVBOY Assessment

Hermes เก่งเรื่อง breadth (22 platforms, 29 providers, research pipeline) แต่ Python = ช้ากว่า Rust สำหรับ local execution

thClaws เก่งเรื่อง depth (native performance, cache optimization, document handling) แต่ ecosystem เล็กกว่า

Claude Code เก่งเรื่อง quality (best model + deep integration) แต่ vendor lock-in

maw-js เก่งเรื่อง control (fleet orchestration, Captain's workflow) แต่ feature set ยังเล็ก

Oracle Ecosystem เก่งเรื่อง knowledge persistence (ψ/ vault survives provider changes, structured learning pipeline, knowledge sovereignty) แต่ inference ยัง Anthropic-dependent + self-learning ยัง semi-manual (/rrr ต้อง trigger เอง ไม่เหมือน Hermes/thClaws ที่ auto)

ไม่มีตัวไหนชนะทุกด้าน — Fleet v3 ใช้ Oracle Ecosystem เป็น knowledge backbone + Claude Code เป็น runtime + maw-js เป็น orchestrator + thClaws สำหรับ mobile + Hermes patterns สำหรับ research
