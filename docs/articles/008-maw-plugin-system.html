maw Plugin System — Architecture & Registry

maw plugin registry · 20 พ.ค. 2026 · ❓ raw

ภาพรวม Plugin System

maw-js ใช้ plugin architecture แบบ modular — ทุก feature ที่ไม่ใช่ core จะอยู่ในรูป plugin ติดตั้ง/ถอดได้

Plugin manager built-in ใน maw-js (เดิมเป็น repo แยก maw-plugin-cli แต่ archived แล้ว)

Plugin Repos

RepoคำอธิบายStatus
maw-plugin-registryCurated registry — manifest source-of-truth สำหรับ maw.soulbrews.studioActive
maw-pluginsInstallable CLI, API, hook pluginsActive
maw-core-pluginsCore plugins: wake, sleep, stop, done, hey, ls, serveActive
maw-incarnation-pluginTemplate plugin — clone เพื่อสร้าง plugin ใหม่Template
maw-cell-pluginExample plugin จาก incarnation templateExample
maw-learnExplore codebase ด้วย parallel AI agents (มาจาก Oracle /learn skill)Active
maw-plugin-cliPlugin manager แยกArchived
maw-commandsCommands แยกArchived (แทนที่ด้วย maw-plugins)

คำสั่งจัดการ Plugin

# List plugins ที่ installed
maw plugin ls

# Install plugin จาก registry
maw plugin install <plugin-name>

# Remove plugin
maw plugin remove <plugin-name>

สร้าง Plugin ใหม่

ใช้ maw-incarnation-plugin เป็น template:

# Clone template
gh repo clone Soul-Brews-Studio/maw-incarnation-plugin my-plugin

# แก้ไข manifest + implement hooks
# publish ไปที่ registry

Core Plugins (มากับ maw-js)

  • wake — start agent ใน tmux window
  • sleep — graceful shutdown agent
  • stop — force stop
  • done — mark task complete
  • hey — ส่ง prompt ไปที่ agent
  • ls — list agents + sessions
  • serve — start API server

  • --prompt-file flag ไม่มีอยู่จริงใน claude CLI (ต้องใช้ --system-prompt-file)
