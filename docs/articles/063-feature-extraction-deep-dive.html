1 — ทำไม Feature Extraction สำคัญ

ML model เก่งได้แค่เท่าข้อมูลที่ใส่เข้าไป — ขยะเข้า = ขยะออก (Garbage In, Garbage Out)

เปรียบเทียบ: สมมติจะจ้างพ่อครัวเก่งที่สุดในโลก แต่ให้วัตถุดิบเน่า — อาหารที่ออกมาก็กินไม่ได้อยู่ดี ML model เหมือนพ่อครัว features คือวัตถุดิบ ต่อให้ algorithm ดีแค่ไหน ถ้า features ห่วย ผลลัพธ์ก็ห่วยตาม

Feature Extraction = กระบวนการเปลี่ยนข้อมูลดิบ (อีเมล, ผลเลือด, พฤติกรรมลูกค้า) ให้เป็น ตัวเลข ที่ model ใช้ตัดสินใจได้

คอมพิวเตอร์ไม่อ่านอีเมลเหมือนคน — มันต้องการตัวเลข เช่น “อีเมลนี้มีลิงก์กี่อัน?” “ผู้ส่งน่าเชื่อถือแค่ไหน?” การเลือก features ที่ถูกต้องคือ 80% ของความสำเร็จ

2 — ผ่าสูตร Spam ทีละตัวเลข

Score = -4.0 + 6.0(1-REP) + 0.25(#RCP) + 0.7(#LINK) + 1.2(#URG) + 0.9(#SPAM)

Score > 0 → SPAM
Score ≤ 0 → NOT SPAM

2.1 — ค่าคงที่ -4.0 คืออะไร?

ค่า -4.0 คือ bias — จุดเริ่มต้นของ model ลองคิดแบบนี้: ถ้าอีเมลมาจากคนน่าเชื่อถือ, ส่งถึงคนเดียว, ไม่มีลิงก์, ไม่มีคำเร่งด่วน, ไม่มีคำ spam ผลรวมฝั่งขวาจะน้อยมาก — หักกับ -4.0 แล้ว score ติดลบ = ไม่ใช่ spam

พูดง่าย ๆ: model ตั้งต้น “เชื่อว่าอีเมลปกติ” ก่อน แล้วให้ features ต่าง ๆ ค่อย ๆ ดัน score ขึ้นไป ถ้าดันข้ามศูนย์ได้ = spam

2.2 — ทำไม REP มีตัวคูณ 6.0 (สูงสุด)?

Featureตัวคูณ (Coefficient)ความหมาย
(1-REP) ความไม่น่าเชื่อถือ6.0Signal แรงที่สุด
#URG จำนวนคำเร่งด่วน1.2แรงรองลงมา
#SPAM จำนวน spam keywords0.9ปานกลาง
#LINK จำนวนลิงก์0.7ปานกลาง
#RCP จำนวนผู้รับ0.25อ่อนที่สุด

ตัวคูณ = น้ำหนักของหลักฐาน — ยิ่งตัวคูณใหญ่ feature นั้นยิ่งมีอิทธิพลต่อการตัดสิน

ทำไม REP สูงสุด? เพราะในโลกจริง ใครส่ง สำคัญกว่า ส่งอะไร — คนแปลกหน้าส่งมาบอก “คุณถูกรางวัล” กับเพื่อนสนิทส่งมาบอก “คุณถูกรางวัล” ต่างกันสิ้นเชิง สูตรนี้สะท้อนสัญชาตญาณมนุษย์ — เช็กผู้ส่งก่อน

สังเกตว่าสูตรใช้ (1-REP) ไม่ใช่ REP ตรง ๆ — เพราะ REP=1.0 หมายถึง “น่าเชื่อถือมาก” แต่เราต้องการ ยิ่งไม่น่าเชื่อถือ ยิ่งได้ score สูง ดังนั้นกลับค่า: REP=0.1 (ไม่น่าเชื่อถือ) → (1-0.1) = 0.9 → 6.0 × 0.9 = 5.4 คะแนน

2.3 — ทำไมโครงสร้าง “ยิ่งเลขสูง = ยิ่ง spam”?

สูตรนี้ออกแบบให้ ทุก feature ดันไปทางเดียวกัน — ยิ่งผู้ส่งไม่น่าเชื่อถือ, ยิ่งผู้รับเยอะ, ยิ่งลิงก์เยอะ, ยิ่งคำเร่งด่วนเยอะ, ยิ่งคำ spam เยอะ → score ยิ่งสูง ทุก feature ช่วยกัน “ดัน” score ข้ามเส้น 0

ถ้า feature ใดไม่น่าสงสัย (เช่น ลิงก์=0) มันก็บวก 0 เข้าไป — ไม่เป็นภาระ ไม่หักคะแนน

2.4 — คำนวณจริง: อีเมลจากเพื่อนสนิท

REP=0.95  #RCP=1  #LINK=0  #URG=0  #SPAM=0

Score = -4.0 + 6.0(1-0.95) + 0.25(1) + 0.7(0) + 1.2(0) + 0.9(0)
     = -4.0 + 6.0(0.05) + 0.25 + 0 + 0 + 0
     = -4.0 + 0.3 + 0.25
     = -3.45 → NOT SPAM ✔

Bias -4.0 แรงมาก features ปกติหมด score ยังติดลบลึก

2.5 — คำนวณจริง: อีเมลหลอกลวง

REP=0.1  #RCP=50  #LINK=5  #URG=3  #SPAM=4

Score = -4.0 + 6.0(1-0.1) + 0.25(50) + 0.7(5) + 1.2(3) + 0.9(4)
     = -4.0 + 6.0(0.9)  + 12.5     + 3.5    + 3.6    + 3.6
     = -4.0 + 5.4        + 12.5     + 3.5    + 3.6    + 3.6
     = +24.6 → SPAM ✔

ทุก feature ช่วยกันดัน — ผู้ส่งไม่น่าเชื่อถือ (+5.4), ส่ง 50 คน (+12.5), ลิงก์ 5 อัน (+3.5) ผลรวมทิ้ง bias -4.0 ไปไกล

3 — Feature Extraction ในโลกจริง (3 ตัวอย่าง)

3.1 — วินิจฉัยโรคจากผลเลือด (Medical Diagnosis)

หมอสั่งตรวจเลือด ผลออกมาเป็นตัวเลขเยอะมาก — แต่ไม่ใช่ทุกตัวเลขจะเกี่ยวข้อง ถ้าจะทำนายว่า คนไข้เป็นเบาหวานหรือไม่ features ที่สำคัญคือ:

Featureตัวอย่างค่าทำไมสำคัญ
Fasting Blood Sugar (FBS)130 mg/dLน้ำตาลในเลือดตอนอดอาหาร — ตัวชี้วัดหลัก
HbA1c7.2%น้ำตาลเฉลี่ย 3 เดือน — เห็นแนวโน้มระยะยาว
BMI28.5ดัชนีมวลกาย — ความอ้วนเพิ่มความเสี่ยง
Age55 ปีอายุมากขึ้น ความเสี่ยงสูงขึ้น
Family History1 (มี) / 0 (ไม่มี)พันธุกรรม — พ่อแม่เป็น ลูกเสี่ยง

สังเกต: สีผม, เบอร์โทร, ชื่อ — ไม่เกี่ยวกับเบาหวานเลย ใส่เข้าไปก็เป็นแค่ noise ทำให้ model สับสน

3.2 — ทำนายราคาบ้าน (House Price Prediction)

จะซื้อบ้าน ราคาขึ้นกับอะไร? features ที่ดี:

Featureตัวอย่างค่าทำไมสำคัญ
พื้นที่ใช้สอย (sq.m.)120บ้านใหญ่ = แพงกว่า (แทบทุกครั้ง)
จำนวนห้องนอน3ห้องมาก = แพงกว่า
ระยะทางถึง BTS (km)0.5ใกล้รถไฟ = แพงขึ้นอย่างชัดเจน
อายุบ้าน (ปี)5บ้านใหม่ = แพงกว่า
ชั้น (กรณีคอนโด)25ชั้นสูง = วิวดี = premium

สิ่งที่ ไม่ควร เป็น feature: สีทาบ้าน, ชื่อเจ้าของ, วันที่ประกาศขาย — ไม่มี signal ต่อราคาจริง

3.3 — ทำนายลูกค้าจะเลิกใช้บริการ (Customer Churn)

บริษัทโทรศัพท์อยากรู้ว่าลูกค้าคนไหนกำลังจะยกเลิก features ที่เป็นสัญญาณเตือน:

Featureตัวอย่างค่าทำไมสำคัญ
จำนวนครั้งโทรเข้า Call Center/เดือน5โทรบ่อย = มีปัญหา = เสี่ยงยกเลิก
ยอดใช้งานเฉลี่ย/เดือน (%การเปลี่ยนแปลง)-40%ใช้น้อยลงเรื่อย ๆ = กำลังหนี
จำนวนเดือนที่ใช้บริการ3ลูกค้าใหม่เปลี่ยนใจง่ายกว่าคนอยู่มา 5 ปี
จำนวนครั้งเปลี่ยนแพ็กเกจ4เปลี่ยนบ่อย = ไม่ satisfied
มีโปรโมชันผูกมัดหรือไม่0 (ไม่มี)ไม่ผูกมัด = ยกเลิกได้ทุกเมื่อ

4 — Dimensions: แต่ละ Feature = 1 มิติ

ทุก feature ที่เพิ่มเข้ามา = เพิ่ม มิติ ให้ข้อมูล 1 อัน

จำนวน Featuresมิติมนุษย์มองเห็นไหม
1 (เช่น น้ำตาลในเลือด)1D — เส้นตรง✔ เห็น
2 (น้ำตาล + BMI)2D — กราฟ x,y✔ เห็น
3 (น้ำตาล + BMI + อายุ)3D — กล่อง x,y,z✔ พอเห็น
5 (spam detector)5D✘ จินตนาการไม่ออก
1,000+ (รูปภาพ)1,000D+✘ ไม่มีทาง

จุดสำคัญ: มนุษย์จินตนาการได้แค่ 2-3 มิติ แต่คอมพิวเตอร์ทำงานกับ 1,000 มิติได้ไม่มีปัญหา — นี่คือเหตุผลที่ ML มีประโยชน์ มันเห็น patterns ใน data ที่คนมองไม่เห็น

ตัวอย่าง: Spam detector มี 5 features = ข้อมูลอยู่ใน 5 มิติ ทุกอีเมลคือ “จุดหนึ่งจุด” ใน space 5 มิตินี้ Model หา “เส้นแบ่ง” ที่แยก spam กับ not-spam ในพื้นที่ 5 มิติ — เหมือนหาเส้นแบ่งบนกราฟ x,y แต่ซับซ้อนกว่า

5 — Feature ดี vs Feature แย่ (Signal vs Noise)

Feature ที่ดี = Signal

  • เปลี่ยนแปลงตามสิ่งที่จะทำนาย — น้ำตาลในเลือดสูง → เบาหวานจริง ๆ มีความสัมพันธ์ชัดเจน
  • วัดได้ ทำซ้ำได้ — “พื้นที่ 120 ตร.ม.” วัดใครก็ได้เลข 120 เหมือนกัน
  • ไม่ซ้ำซ้อนกับ features อื่น — ถ้าใส่ทั้ง “พื้นที่ (ตร.ม.)” และ “พื้นที่ (ตร.ฟุต)” = ข้อมูลเดียวกัน 2 ครั้ง ไม่ช่วยอะไร

Feature ที่แย่ = Noise

  • ไม่เกี่ยวกับสิ่งที่ทำนาย — สีทาบ้านไม่บอกราคาบ้าน เบอร์โทรไม่บอกว่าเป็นเบาหวาน
  • สุ่มเกินไป — เลขที่บ้านอาจ “บังเอิญ” ตรงกับราคาในข้อมูล training แต่ไม่มี logic อะไร model จะ “จำ” pattern ปลอมนี้ไป → ทำนายผิดกับข้อมูลใหม่ (overfitting)
  • เปลี่ยนแปลงตลอดเวลาแบบไม่มี pattern — เช่น timestamp ตอนกดบันทึก ไม่มี signal

ทดสอบง่าย ๆ: “ถ้า feature นี้เปลี่ยน ผลลัพธ์ควรเปลี่ยนด้วยไหม?”

Featureผลลัพธ์เปลี่ยนไหม?ดี/แย่
น้ำตาลในเลือด 90 → 200✔ ใช่ ความเสี่ยงเบาหวานสูงขึ้นชัดGood
เบอร์โทรคนไข้เปลี่ยน✘ ไม่ ไม่เกี่ยวกับเบาหวานBad
ระยะถึง BTS 0.5 → 15 km✔ ใช่ ราคาบ้านต่างกันมากGood
สีบ้าน ขาว → ฟ้า✘ ไม่ ไม่กระทบราคาจริงBad

6 — สรุป: Feature Extraction ใน 1 ประโยค

Feature Extraction คือศิลปะของการเลือกว่า จะบอกอะไรกับ model — เลือกดี model เก่ง เลือกห่วย model โง่ ไม่ว่า algorithm จะดีแค่ไหน

แนวคิดจำแบบนี้
Garbage In = Garbage Outวัตถุดิบเน่า = อาหารเน่า ต่อให้พ่อครัวเก่ง
Coefficient = น้ำหนักหลักฐานตัวคูณใหญ่ = feature นั้นสำคัญมาก
Bias = จุดเริ่มต้น-4.0 = “เชื่อว่าปกติก่อน จนกว่าหลักฐานจะพิสูจน์เป็นอื่น”
Dimensions = จำนวน features5 features = 5 มิติ คอมเห็นได้หมด คนเห็นได้แค่ 3
