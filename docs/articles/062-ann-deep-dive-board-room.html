1 — จาก y = mx + c สู่ Neuron

Linear Regression คือสูตรง่ายที่สุดในโลก:

y = mx + c

m = ความชัน (slope)
c = จุดตัดแกน y (intercept)

Neuron 1 ตัว ทำเหมือนกันเป๊ะ — แค่เปลี่ยนชื่อ: m เรียก weight (w), c เรียก bias (b)

y = w · x + b     ← Neuron 1 input

ถ้ามีหลาย input ก็คูณแต่ละตัวด้วย weight ของมัน แล้วรวมกัน:

y = w1·x1 + w2·x2 + w3·x3 + b

อุปมา: กรรมการตัดสินการแข่งขัน

คิดว่า neuron เป็น กรรมการ ที่ให้คะแนนผู้เข้าแข่งขัน:

  • แต่ละเกณฑ์ (x) มี น้ำหนักความสำคัญ (w)
  • คะแนนรวม = ผลรวมถ่วงน้ำหนัก
  • เช่น เกณฑ์ "ความคิดสร้างสรรค์" สำคัญมาก (w=0.5) แต่ "ความเร็ว" สำคัญน้อย (w=0.1)

2 — Board Room ANN — คำนวณด้วยมือ

3 inputs → 2 hidden neurons → 3 outputs

ทุก input ต่อกับทุก hidden neuron, ทุก hidden neuron ต่อกับทุก output — เรียกว่า fully connected

2.2 — Step 1: Input → Hidden Layer

กำหนด weights ของ hidden neurons:

w_ROIw_Alignw_Riskbias
H10.60.3-0.50.1
H20.20.50.8-0.2

คำนวณ H1:

H1 = (0.6 × 0.8) + (0.3 × 0.7) + (-0.5 × 0.3) + 0.1
   =    0.48     +    0.21     +   (-0.15)    + 0.1
   = 0.64

H1 = 0.64 — สังเกต: w_Risk เป็นลบ → Risk สูง = ดึงคะแนนลง (สมเหตุสมผล)

คำนวณ H2:

H2 = (0.2 × 0.8) + (0.5 × 0.7) + (0.8 × 0.3) + (-0.2)
   =    0.16     +    0.35     +    0.24     - 0.2
   = 0.55

H2 = 0.55 — H2 ให้น้ำหนัก Risk สูง (w=0.8) = neuron ตัวนี้ "ตรวจจับความเสี่ยง"

2.3 — Step 2: Hidden → Output Layer

H1 (0.64) กับ H2 (0.55) กลายเป็น input ของ 3 output neurons:

w_H1w_H2bias
Approve0.9-0.40.1
Delay0.30.30.0
Reject-0.50.70.2
Approve = (0.9 × 0.64) + (-0.4 × 0.55) + 0.1
        =    0.576    +   (-0.22)    + 0.1
        = 0.456

Delay   = (0.3 × 0.64) + (0.3 × 0.55) + 0.0
        =    0.192    +    0.165    + 0.0
        = 0.357

Reject  = (-0.5 × 0.64) + (0.7 × 0.55) + 0.2

3 — Activation Function — ประตูที่ทำให้ Network ฉลาดขึ้น

ตัวอย่างข้างบนเป็นแค่การบวกคูณ (linear) — ไม่ว่าจะมีกี่ layer ก็ยุบรวมเป็นสูตร linear ได้เสมอ เหมือนกรรมการที่ทำได้แค่ ลากเส้นตรง

Activation function เพิ่ม "กฎ if-then" หลังการคำนวณ → network เรียนรู้ pattern ที่ซับซ้อนได้

ReLU — วาล์วน้ำทางเดียว

ReLU(x) = x   ถ้า x > 0
           0   ถ้า x ≤ 0

เหมือนวาล์วน้ำ: น้ำไหลได้ทางเดียว (บวก)
ค่าลบถูกปิดกั้นหมด → 0

ตัวอย่าง:
  ReLU(0.64)  = 0.64   (ผ่าน)
  ReLU(-0.3)  = 0      (ปิดกั้น)

Sigmoid — เกจวัดความมั่นใจ

Sigmoid(x) = 1 / (1 + e^(-x))

บีบทุกค่าให้อยู่ระหว่าง 0 ถึง 1
เหมือนเกจวัดความมั่นใจ:
  ค่าลบมาก → เข้าใกล้ 0 (ไม่มั่นใจ)
  ค่าบวกมาก → เข้าใกล้ 1 (มั่นใจมาก)

ตัวอย่าง:
  Sigmoid(0.64) = 1/(1 + e^(-0.64)) = 0.655
  Sigmoid(-2.0) = 1/(1 + e^(2.0))   = 0.119
  Sigmoid(5.0)  = 1/(1 + e^(-5.0))  = 0.993

ถ้าใส่ ReLU ใน Board Room: H1=0.64 ผ่าน (เป็นบวก), H2=0.55 ผ่าน — ในกรณีนี้ผลเหมือนเดิมเพราะทั้งคู่เป็นบวก แต่ถ้า H1 ออกมาติดลบ ReLU จะตัดเป็น 0 → neuron ตัวนั้น "ปิด" → ข้อมูลไหลผ่านเฉพาะ path ที่มีค่า

4 — ทำไมต้อง “หลาย Layer”

Layersเรียนรู้อะไรได้อุปมา
1 layerเส้นตรง — pattern ง่ายๆร่างภาพด้วยไม้บรรทัด
2 layersเส้นโค้ง + การรวม patternร่างภาพด้วยไม้โค้ง (French curve)
หลาย layers (Deep)ใบหน้า, ภาษา, เสียง, ดนตรีภาพ HD จากกล้องโปร

Board Room ของเรามี 1 hidden layer — เพียงพอสำหรับ "ถ้า ROI สูง + Risk ต่ำ → Approve" แต่ถ้าจะตัดสินใจซับซ้อนกว่านี้ (เช่น วิเคราะห์เอกสารโครงการ + ประวัติทีม + สภาพตลาด) ต้องเพิ่ม layers อีกหลายชั้น

1 filter  = ภาพร่างคร่าวๆ
3 filters = เห็นรูปร่าง
10 filters = เห็นรายละเอียด
100+ filters (Deep Learning) = เห็นทุกอย่าง

5 — Weight Matrix — เลขกณิตที่อยู่เบื้องหลัง

Weights ทั้งหมดจาก Input → Hidden เขียนเป็น matrix (ตารางตัวเลข) ได้:

    Input vector       Weight matrix W         Bias vector
                     (2 rows × 3 cols)

                    | w_ROI   w_Align  w_Risk |
                    |  0.6      0.3     -0.5  |   H1 row
                    |  0.2      0.5      0.8  |   H2 row

    [ROI, Align, Risk] × W^T + [b1, b2] = [H1, H2]

คำนวณทีละขั้น (matrix multiplication):

Step 1: แถว H1 × input column

  | 0.6  0.3  -0.5 |   | 0.8 |         | 0.54 |
  | 0.2  0.5   0.8 | × | 0.7 |    =    | 0.75 |
                        | 0.3 |

  H1 = (0.6×0.8) + (0.3×0.7) + (-0.5×0.3) = 0.48 + 0.21 - 0.15 = 0.54
  H2 = (0.2×0.8) + (0.5×0.7) + (0.8×0.3)  = 0.16 + 0.35 + 0.24 = 0.75

Step 2: บวก bias

  H1 = 0.54 + 0.1  = 0.64  ✔
  H2 = 0.75 + (-0.2) = 0.55  ✔

ตัวเลขตรงกับที่คำนวณด้วยมือในข้อ 2 — matrix multiplication ทำเหมือนกันแค่เขียนสั้นกว่า

ข้อดีของ matrix form: คอมพิวเตอร์คำนวณ matrix multiplication เร็วมาก (GPU ทำได้พันล้าน operations ต่อวินาที) — นี่คือเหตุผลที่ Deep Learning ต้องใช้ GPU

Key Insight

"การเรียนรู้" ของ AI = ปรับตัวเลขใน weight matrix จนผลลัพธ์ถูกต้อง

Board Room ANN มี (3×2) + (2×3) = 12 weights + 5 biases = 17 parameters

ChatGPT/Claude มี หลายร้อยพันล้าน parameters — แต่หลักการเดียวกัน: คูณ, บวก, ผ่าน activation, ส่งต่อ — ซ้ำหลาย layer จนได้คำตอบ
