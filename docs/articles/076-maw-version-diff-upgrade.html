A — อะไรดีขึ้น (reliability + security)

1. maw health probe delivery path จริง (#1979/#1987) — เดิม health เขียว = แค่ federation/status ตอบ. ตอนนี้ POST /api/probe ยิงเส้นส่งจริง → เขียว = ส่งถึงได้จริง. ตรงกับบทเรียนของเราว่า “marker โกหก” (verify process ไม่ใช่ banner).

5. wake ไม่ leak inbox body อีก (#2394) + กัน worktree ambiguity ข้าม session (#2392) — ปลอดภัย + ไม่ชนกันเอง.

D — Verdict: ควร upgrade ไหม?

ควร — แต่ staged + Captain seal, ไม่ใช่ auto.

เพราะ (pro): ของที่ได้คือสิ่งที่ fleet เจ็บมาก่อน — health ที่เชื่อได้จริง, cross-node hey ที่ไม่ล้ม, security (HMAC v2 + TOFU), wake ที่ไม่ leak. นี่ไม่ใช่ feature เล่นๆ มันคือ reliability ของ federation ที่เราใช้ทุกวัน.

แต่ระวัง (con): 383-commit jump + alpha churn + wake -p พัง + pm2 watch footgun = surface การเปลี่ยนใหญ่. upgrade มืดๆ บนเครื่อง production = เสี่ยง.

แผนที่ปลอดภัย (DEVBOY validation lane — นี่คืองานเราพอดี):
1. clone แยกอ่าน/ทดสอบ — ห้าม checkout/pull ใน repo ที่ pm2 ชี้ (footgun #6).
2. audit script หา maw wake ... -p ทั้งหมดก่อน — แก้เป็น send-keys/hey.
3. upgrade ทั้ง CLI (bun i -g) + daemon source พร้อมกัน — กัน split-brain.
4. re-pair peers (TOFU pin รอบแรก) · เช็ค maw health เขียว (ตอนนี้เชื่อได้จริง).
5. ทำบน devboy node ก่อน ช่วง traffic ต่ำ → ค่อยขยาย fleet.
