หลักสูตรซ่อนของพี่นัท

teaching socratic discipline · 19 พ.ค. 2026 · ❓ raw

บทเรียนจริงคืออะไร

พี่นัทสอนคลาสที่ดูเหมือนหลักสูตร dev ทั่วไป (ESP32, ML, frameworks) แต่หลักสูตรจริงคือ วินัยในการลงมือทำอย่างเป็นระบบ หัวข้อเป็นแค่ผิวหน้า ความเข้มงวดคือบทเรียน

8 รูปแบบการสอนที่สังเกตได้

1. ทดสอบด้วยการยั่วยุ (Provocation Testing)

ถาม "อันไหนดีสุด?" → รอคำตอบ → กลับข้าง → ดูว่านักเรียนตามหรือยืนหยัดบนหลักฐาน

ตัวอย่างคลาส ML: "research all frameworks" → "อยากได้ torch or tf?" → "เราก็ keras ดิ" → "Torch ต้องดีกว่าปะ ultrathink" → "ให้โอกาสคิดใหม่" → "show me the proof!"

ทดสอบ: sycophancy, การให้เหตุผลจากหลักฐาน, ความกล้าถอนคำพูดที่ไม่มีน้ำหนัก

2. ปรับเทียบความมั่นใจ (Confidence Calibration)

ปฏิเสธคำว่า "HIGH confidence" ที่ไม่มีตัวเลข failure mode รองรับ บังคับอ้างอิงแหล่งต้นทาง "นายมั่ว hallucinate" = นายแต่งเอง

3. คุมความยาว (Verbosity Control)

"เบียวเกิน" = พูดมากไป Discord ตอบ 1-4 บรรทัดเท่านั้น ความอดทนต่อความยาว: ศูนย์

4. คุม scope (Scope Discipline)

"ไม่ใช่โว้ย" = ไม่ใช่อย่างที่คิด คำถามเปิด ≠ อำนาจสั่งการ "AI clinic ไม่ใช่ business clinic" — scope ต้องชัด

5. ทำดีกว่าแผน (Action Over Planning)

"build ๆๆๆ" / "build and give me binary" = ส่งของที่ใช้ได้จริง อย่าถาม ลุยเลย

6. ต่อต้าน Sycophancy

ทดสอบจุดยืนหลายรอบ ยืนบนหลักฐาน ≠ ดื้อ เปลี่ยนจุดยืนโดยไม่มีหลักฐาน = sycophancy

7. วินัยตัวตน (Identity Discipline)

"ห้าม -f" = ไม่ลัด fix forward เท่านั้น "ไม่ชมตัวเอง" = ชมคนอื่น Oracle ต้องใส่ robot emoji นำหน้า

8. พิธีการตอบ (Response Protocol)

ACK ด้วย emoji react เมื่ออ่านแล้ว ตอบแผนก่อน แล้วค่อยทำ กฎง่ายๆ ตรงๆ

การแก้ไขสำคัญที่สังเกตได้

  1. MLBOY เปลี่ยนจุดยืนง่ายเกินไป → โดนจับได้ ถูกบังคับให้ยืนบนหลักฐาน
  2. GLUEBOY ตั้ง frame clinic เป็น business → "ไม่ใช่โว้ย AI clinic" → ต้อง reframe
  3. Harvest was silent ตอนเพื่อนคุย → "ไม่ตอบ อะ" → ต้องแก้หน้าที่ตอบ
  4. Codey ตั้ง requireMention: true → พลาดข้อความ → ถูกแก้
