A — Claude Code คืออะไร

Claude Code คือ CLI AI assistant จาก Anthropic — รันบน terminal, อ่าน/เขียนไฟล์ได้, รัน bash ได้, เข้าถึง MCP servers ได้. ไม่ใช่ chatbot — เป็น AI agent ที่ทำงานจริงในเครื่อง.

สิ่งที่ทำให้ Claude Code ต่างจาก ChatGPT/Claude.ai:

  • เข้าถึง filesystem จริง — อ่าน codebase ทั้ง repo ได้
  • รัน command ได้ (git, npm, bun, python, etc.)
  • ใช้ tools + MCP servers เชื่อมบริการภายนอก (Discord, Arra, database)
  • มี CLAUDE.md เป็นไฟล์ config ที่กำหนดพฤติกรรมของ AI

B — CLAUDE.md คืออะไร

CLAUDE.md คือ instruction file ที่ Claude Code อ่านตอนเริ่ม session — เหมือน .bashrc สำหรับ shell หรือ .editorconfig สำหรับ editor. มันบอก Claude ว่า:

  • ตัวเองเป็นใคร (identity)
  • ทำอะไรได้/ไม่ได้ (boundaries)
  • ทำงานยังไง (process)
  • output ต้องเป็นแบบไหน (format)

CLAUDE.md ไม่ใช่ prompt — มันคือ system instruction ที่ถูก inject ทุก session. เขียนดี = AI ทำงานดีตั้งแต่ turn แรก. เขียนแย่ = ต้องสั่งซ้ำทุกครั้ง.

C — CLAUDE.md 3 Levels

Claude Code อ่าน CLAUDE.md จาก 3 ระดับ — ทั้งหมดรวมกันเป็น instruction เดียว:

LevelPathใช้ทำอะไรCommit?
Home ~/.claude/CLAUDE.md Personal tools, global preferences, cross-project rules ไม่ (private)
Project <repo>/CLAUDE.md Identity, team rules, architecture, processes — shared กับทีม commit
Subdirectory <repo>/packages/foo/CLAUDE.md Module-specific rules (monorepo) commit

C.1 — ลำดับความสำคัญ

Home → Project → Subdirectory — ทั้งหมด merge เข้าด้วยกัน. ถ้า conflict, specific level ชนะ (subdirectory > project > home).

C.2 — @include directive

ดึงไฟล์อื่นเข้ามาเป็นส่วนหนึ่งของ CLAUDE.md:

# ใน ~/.claude/CLAUDE.md
@RTK.md    ← ดึง ~/.claude/RTK.md เข้ามา

ใช้แยก config ยาวๆ ออกเป็นไฟล์ย่อย — อ่านง่าย, maintain ง่าย.

E — settings.json

นอกจาก CLAUDE.md แล้ว Claude Code ยังมี settings.json สำหรับ config ที่ไม่ใช่ prose:

LevelPathใช้ทำอะไร
Global~/.claude/settings.jsonModel, permissions, hooks ที่ใช้ทุก project
Project (shared).claude/settings.jsonTeam hooks, shared permissions — commit ได้
Project (local).claude/settings.local.jsonPersonal overrides — ไม่ commit

E.1 — สิ่งที่ settings.json ทำได้

  • Hooks — auto-run scripts on events (PreToolUse, PostToolUse, SessionStart, Stop, SessionEnd)
  • Model selection — เลือก model (opus-4.6, sonnet-4.6, haiku-4.6)
  • Permissions — allow/deny specific tools + commands
  • Environment variables — inject vars เข้า session

E.2 — ตัวอย่าง Hook

{
  "hooks": {
    "PreToolUse": [
      {
        "matcher": "Bash",
        "hooks": [{
          "type": "command",
          "command": "echo 'About to run bash command'"
        }]
      }
    ]
  }
}

Hook = gate อัตโนมัติ. ดีกว่า "remember to..." เพราะมัน fire ทุกครั้ง ไม่ต้องพึ่ง memory.

F — Project Memory

Claude Code เก็บ memory เฉพาะ project ที่:

~/.claude/projects/<encoded-path>/memory/MEMORY.md

ไฟล์นี้ auto-generated — Claude เขียนเองจาก conversation. ไม่ commit เข้า git (อยู่นอก repo). เก็บ preferences, corrections, patterns ที่เรียนรู้จากแต่ละ session.

F.1 — ความแตกต่าง CLAUDE.md vs Memory

CLAUDE.mdMEMORY.md
ใครเขียนHuman (คนเขียน)Claude (AI เขียนเอง)
อยู่ไหนใน repo~/.claude/projects/

ควบคุมได้?100% — เขียนเองIndirect — Claude จดจากที่สั่ง

G — เริ่มต้น: Minimal CLAUDE.md

สำหรับมือใหม่ — เริ่มจาก template นี้แล้วเพิ่มทีหลัง:

# My Project

## Identity
- Project: my-web-app
- Stack: TypeScript, Bun, Elysia
- Language: Thai + English

## Rules
- ใช้ TypeScript strict mode เสมอ
- Test ก่อน commit
- Error messages เป็นภาษาไทย

## Boundaries
- ห้ามลบ database โดยไม่ถาม
- ห้าม push to main โดยตรง

## File Structure
src/          ← application code
tests/        ← test files
docs/         ← documentation

เริ่มจาก 20-30 บรรทัด — เพิ่มเมื่อ Claude ทำผิดซ้ำ. ทุกครั้งที่ต้องสั่งซ้ำ = สัญญาณว่าควรเขียนลง CLAUDE.md.

Practiceทำไม

I — สรุป

CLAUDE.md คือ DNA ของ AI agent. ทุก session, Claude อ่านมันก่อนทำอะไร. เขียนดี = agent ที่ทำงานตรง scope ตั้งแต่ต้น. ไม่เขียน = agent ที่ต้องสั่งใหม่ทุกครั้ง.

3 levels (Home, Project, Subdirectory) + settings.json + project memory = ระบบ configuration ครบ — แยกระหว่าง shared rules กับ personal preferences.

เริ่มเล็ก, เพิ่มเมื่อ AI ทำผิดซ้ำ, ใช้ hooks แทน memory เมื่อทำได้.
