Team-Tile คืออะไร

Team-Tile Bootstrap = pattern สำหรับ spawn N parallel claude.exe teammates ใน tmux panes โดยใช้ เฉพาะ maw verbs + Claude Code tools ไม่ต้อง raw tmux split-window

Validated live ใน digger-oracle fleet 2026-05-20 โดย P'Nat. Source: gist

Verb Chain

maw tile N → tmux select-layout → TeamCreate → SendMessage × N
           → wait → SendMessage shutdown × N → TeamDelete → kill-pane

หลัง maw-js #1837 — collapsed เหลือ 1 verb ต่อ pane:

maw tile 1 --path <repo> --cmd "env ... claude.exe --agent-id ..."

Security Model

  • --dangerously-skip-permissions propagate จาก lead ไปทุก teammate
  • ทุก teammate run เป็น same Unix user — ไม่มี per-teammate sandbox
  • Teammate สามารถ write file ใดก็ได้ที่ user เป็นเจ้าของ
  • Teammate สามารถ inject message เข้า inbox teammate อื่น (Seam 4)
  • Security boundary = OS user isolation เท่านั้น

maw-js #1837 — Evolution

ก่อน #1837 ต้อง 2 verbs ต่อ pane (timing race ระหว่าง tile กับ run):

maw tile 2
maw run 50-digger:2.1 "cd /opt/Code/foo && env ... claude.exe ..."

หลัง #1837 — atomic spawn 1 verb:

maw tile 1 --path /opt/Code/foo --cmd "env ... claude.exe ..."
Key improvement: ไม่มี timing gap ระหว่าง pane creation กับ command execution — ถ้า tile fail ไม่มี orphaned pane, ถ้า succeed command รันทันที.

DEVBOY Takeaways

  1. team-tile complement Agent() ไม่ใช่ replacement — ใช้ team-tile เมื่อต้อง cross-repo, persistent, bidirectional; ใช้ Agent() เมื่อ same-repo ephemeral result-only
  2. maw tile คือ new primitive ที่ DEVBOY ยังไม่เคยมี — extends conceptual knowledge จาก article 003 สู่ implementation level
  3. Seam 6 = ต้องจำ — tmux kill-pane เสมอหลัง shutdown_approved
  4. Security concern — --dangerously-skip-permissions hardcoded ใน bootstrap.ts = ทุก teammate ทำอะไรก็ได้
  5. Lab experiment required — run ψ/lab/team-tile-bootstrap/ บน DO server ก่อน promote to solid
  6. Pre-bud potential — team-tile สามารถเป็น mechanism สำหรับ test bud candidates ก่อน maw bud
