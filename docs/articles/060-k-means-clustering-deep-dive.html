1 — What is Clustering?

Imagine you dump 100 mixed candies on a table. Nobody tells you what the groups are. You just naturally start sorting: red ones here, green ones there, yellow in the corner. You are clustering — finding groups that share something in common, without any labels or instructions.

That is exactly what unsupervised learning does. The algorithm looks at the data, finds natural groupings, and assigns each data point to a group. No teacher, no answer key.

Supervised Learning:   "This candy is RED" (you have labels)
Unsupervised Learning: "These candies look similar" (no labels — find groups yourself)

K-Means is the most common clustering algorithm. The “K” = how many groups you want. “Means” = the algorithm uses averages (means) to find the center of each group.

3 — SSE (Sum of Squared Errors)

SSE answers one question: “How spread out are the points from their center?”

SSE = sum of (distance from each point to its centroid)²

Lower SSE = tighter groups = better clustering
Higher SSE = looser groups = worse clustering

For our final clusters:

Cluster 1 (centroid 2.33, 5.00):
  A: (2-2.33)² + (3-5.00)² = 0.11 + 4.00 = 4.11
  B: (3-2.33)² + (4-5.00)² = 0.45 + 1.00 = 1.45
  E: (2-2.33)² + (8-5.00)² = 0.11 + 9.00 = 9.11

Cluster 2 (centroid 6.67, 8.00):
  C: (8-6.67)² + (7-8.00)² = 1.77 + 1.00 = 2.77
  D: (9-6.67)² + (8-8.00)² = 5.43 + 0.00 = 5.43
  F: (3-6.67)² + (9-8.00)² = 13.47 + 1.00 = 14.47

Total SSE = 4.11 + 1.45 + 9.11 + 2.77 + 5.43 + 14.47 = 37.33

4 — The Elbow Method: “How Many Groups?”

The hardest question in K-Means: what K should I pick? The Elbow Method is a visual trick.

Run K-Means with K = 1, 2, 3, 4, 5. Record the SSE each time:

K=1:  SSE = 95.3   (everything in one blob)
K=2:  SSE = 37.3   (big drop!)
K=3:  SSE = 12.0   (another big drop!)
K=4:  SSE = 10.1   (tiny improvement)
K=5:  SSE =  8.8   (barely moved)

SSE
 |
95|  *
   |
   |
37|      *
   |
12|          *
10|              *
 9|                  *
   +--+--+--+--+--+--
      1  2  3  4  5   K

The “elbow” is where the line bends — SSE stops dropping fast and starts to plateau. Here the elbow is at K = 3. Going from 1 to 3 gives huge improvement. Going from 3 to 5 barely helps. So K = 3 is the sweet spot.

Rule of thumb: more groups always lowers SSE (at K = N, SSE = 0 because each point is its own cluster). The elbow tells you where the “return on investment” of adding groups runs out.

5 — When K-Means Fails

K-Means assumes clusters are round (spherical) and roughly the same size. When real data breaks these assumptions, it struggles:

5.1 — Non-spherical clusters

Reality: two crescent moons     K-Means sees:

  ** ***                         [---Cluster 1---]
 *       *                       [---Cluster 2---]
  ** ***
       ** ***                    K-Means cuts straight
      *       *                  through the crescents.
       ** ***                    It cannot wrap around shapes.

5.2 — Different-sized clusters

  Small tight        LARGE spread cluster
  cluster ****       *    *      *
   (50 pts) **       *  *    *  *   *
            **       *      *     *
                     * *  *   *  * *
                        (500 pts)

K-Means steals points from the big cluster to “balance” the small one.

5.3 — Different density clusters

One cluster is packed tight, another is spread thin. K-Means treats them the same because it only cares about distance, not density.

5.4 — Sensitive to starting position

Same data, different random centroids → different final clusters.

Run 1: centroids start at (1,1) and (2,2) → bad result (local minimum)
Run 2: centroids start at (1,1) and (9,9) → good result

Fix: run K-Means 10-20 times with different random starts.
     Keep the result with the lowest SSE.
     (This is what “K-Means++” initialization improves.)

Key Takeaways

#Takeaway
1Clustering = finding groups without labels. Like sorting candies by color without anyone telling you the categories.
2K-Means repeats 3 steps: assign to nearest centroid → recalculate centroid → repeat until stable.
3SSE measures cluster quality. Lower = tighter = better.
4Elbow Method picks K. Plot SSE vs K — the bend in the curve is your answer.
5K-Means fails on non-round, uneven clusters and is sensitive to random starting points. Run it multiple times.
