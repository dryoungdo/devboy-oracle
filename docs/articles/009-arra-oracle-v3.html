Arra Oracle คืออะไร

Arra Oracle v3 คือ MCP memory layer + semantic knowledge base — ออกแบบมาให้ AI agent จัดเก็บ, index, และ retrieve ความรู้ขององค์กร ผ่าน hybrid search

Version ปัจจุบัน: v26.4.19-alpha.7 | Docs: 20,672 รายการ

Hybrid Search Architecture

Arra ใช้ search 2 แบบรวมกัน:

MethodEngineจุดเด่น
FTS5SQLite Full-Text Search 5เร็ว, keyword-based, sanitize input ป้องกัน parse error
Vector SearchChromaDB / LanceDB / sqlite-vec / QdrantSemantic similarity, embedding-based

Hybrid mode merge ผลจากทั้งสองด้วย re-ranking — ถ้า vector ไม่พร้อมจะ fallback เป็น FTS5-only

Embedding Models

  • bge-m3 (default) — multilingual ไทย-อังกฤษ
  • nomic — general purpose
  • qwen3 — 4096 dimensions

Technical Stack

Runtime:    Bun ≥1.2 (TypeScript)
Database:   SQLite + FTS5, Drizzle ORM
Vectors:    LanceDB (default), multi-backend
API:        Hono HTTP server on port 47778
Protocol:   MCP for Claude integration
Install:    bunx (GitHub packages)

Key Concepts

  • arra_learn — เพิ่ม pattern/learning เป็น markdown ใน ψ/memory/learnings/ auto-index พร้อม concept tags
  • arra_trace — capture discovery sessions เป็น queryable dig points (files, commits, issues) พร้อม parent-child linking
  • Supersede Pattern — "Nothing is Deleted" — mark เป็น outdated แทนที่จะลบ เก็บ lineage ไว้
  • Forum (arra_thread) — discussion threads พร้อม Oracle auto-response จาก knowledge base
