Part B — Code Excerpts and Mechanisms

B1. MAW_CLI guard at server.ts bottom

Source: ~/.local/share/maw-js/alpha/src/core/server.ts.

export async function startServer(port = +(process.env.MAW_PORT || loadConfig().port || 3456)) {
  // ... Bun.serve(...) binds the port inside here ...
  return server;
}

// Auto-start unless imported by CLI (CLI sets MAW_CLI=1)
if (!process.env.MAW_CLI) {
  startServer();
}

The guard is correct for CLI imports. The failure was launch context: pm2 is not the CLI, but it inherited the CLI marker. Fix the environment boundary, not the guard.

B2. Restore wrapper removes MAW_CLI before pm2 start

Source: glueboy-oracle/scripts/restore-maw.sh, commit 1debff5.

log "starting pm2 maw from $MAW_SOURCE with MAW_CLI removed from environment"
if ! (
  cd "$MAW_SOURCE"
  env -u MAW_CLI pm2 start ecosystem.config.cjs --only maw
); then
  fail "pm2 start maw failed"
  exit 1
fi

Important detail: env -u MAW_CLI wraps the pm2 command itself. That prevents pm2 from storing the poisoned environment for future restarts.

B3. --force bypasses the local idle guard

Source: ~/.local/share/maw-js/alpha/src/commands/shared/comm-send.ts. The local tmux path checks agent process and prompt-line idle state only when force is false.

if (result?.type === "local" || result?.type === "self-node") {
  const target = await resolveOraclePane(result.target);
  if (!force) {
    const cmd = await getPaneCommand(target);
    const isAgent = isAgentCommand(cmd);
    if (!isAgent) {
      if (logQueuedInbox(await writeReceiverInbox(target), target, `pane not running an agent (${cmd})`)) return;
      // ...
    }

    let idleCheck = await checkPaneIdle(target);
    if (!idleCheck.idle) {
      await Bun.sleep(500);
      idleCheck = await checkPaneIdle(target);
      if (!idleCheck.idle) {
        if (logQueuedInbox(await writeReceiverInbox(target), target, `pane not idle: ${idleCheck.lastInput.slice(0, 60)}`)) return;
        // ...
      }
    }
  }
  await sendKeys(target, outboundMessage);
  // ...
}

For Codex panes, that idle detector becomes overprotective because the status line is constantly active. --force is therefore not "reckless force" in this context; it is the documented path for a known false positive.

B4. The idle detector's shape

The detector inspects the last visible prompt line. It was designed to avoid clobbering human input, not to classify Codex TUI state.

export async function checkPaneIdle(target: string): Promise<{ idle: boolean; lastInput: string }> {
  const content = await capturePane(target, 5, host);
  const lines = content.split("\n").filter(l => l.trim());
  const lastLine = lines.at(-1) ?? "";
  const clean = lastLine.replace(/\x1b\[[0-9;]*[mGKHFJA-Z]/g, "").replace(/\r/g, "");

  if (/[#$%>❯»]\s*$/.test(clean)) return { idle: true, lastInput: "" };
  const notIdleMatch = clean.match(/[#$%>❯»]\s+(\S.*)$/);
  if (notIdleMatch) return { idle: false, lastInput: notIdleMatch[1] };
  return { idle: true, lastInput: "" };
}

B5. Symlink-aware runtime checks use readlink -f

Source: glueboy-oracle/scripts/check-glueboy-config.sh. Bun installs and links can be relative symlink chains, so comparing raw paths is misleading. Resolve both ends first.

# bun installs maw via a relative symlink through node_modules; compare the
# fully-resolved paths rather than the raw (relative) link target.
maw_resolved="$(readlink -f "$MAW_BIN" 2>/dev/null || true)"
maw_cli_resolved="$(readlink -f "$MAW_CLI" 2>/dev/null || true)"
if [[ -n "$maw_resolved" && "$maw_resolved" == "$maw_cli_resolved" ]]; then
  pass "maw CLI points at stable alpha source"
else
  fail "maw CLI link is unexpected: ${maw_resolved:-not resolvable}; expected $MAW_CLI"
fi

maw_server_resolved="$(readlink -f "$MAW_SERVER" 2>/dev/null || echo "$MAW_SERVER")"
pm2_maw_script="$(pm2 describe maw 2>/dev/null | rg -o '/[^[:space:]│]+server\.ts' | head -1 | tr -d ' ')"
pm2_maw_script_resolved="$(readlink -f "$pm2_maw_script" 2>/dev/null || echo "$pm2_maw_script")"

Lesson: in a Bun-linked toolchain, which maw is not enough. Use readlink -f to identify the actual file pm2 and the shell are executing.
