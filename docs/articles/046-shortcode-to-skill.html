จาก Shortcode สู่ Skill — สร้าง Slash Command ของตัวเอง

skills shortcode slash-command create-shortcut automation skill-md · CCC Academy Lesson 2 ✔ solid

A — Skill คืออะไร

Skill คือ markdown file (SKILL.md) ที่ Claude Code อ่านเป็น slash command. พิมพ์ /rrr — Claude อ่าน SKILL.md ของ rrr แล้วทำตาม instructions ข้างใน.

Skill ไม่ใช่ code — เป็น natural language instructions ที่ Claude ตีความแล้วทำ. เขียนเป็น markdown ธรรมดา — ไม่ต้องเป็น programmer ก็สร้างได้.

/bye        →  ~/.claude/skills/bye/SKILL.md        (20 บรรทัด)
/forward    →  ~/.claude/skills/forward/SKILL.md    (271 บรรทัด)
/rrr        →  ~/.claude/skills/rrr/SKILL.md        (267 บรรทัด + DEEP.md + TEAMMATE.md)

B — Skill อยู่ที่ไหน

LevelPathScope
Project .claude/skills/<name>/SKILL.md ใช้ได้เฉพาะใน repo นี้ — commit ได้, ทีมใช้ร่วม
Global ~/.claude/skills/<name>/SKILL.md ใช้ได้ทุก project — personal, ไม่ commit

Project skills ชนะ global ถ้าชื่อซ้ำ — เหมือน CLAUDE.md ที่ specific level ชนะ.

C — โครงสร้าง SKILL.md

---
name: my-skill
description: "ทำ X เมื่อ user พิมพ์ /my-skill"
argument-hint: "[--fast | --deep]"
---

# /my-skill

คำอธิบายสั้นๆ ว่า skill นี้ทำอะไร.

## Usage

```
/my-skill              # default mode
/my-skill --fast       # quick version
/my-skill $ARGUMENTS   # pass custom args
```

## Step 1: ทำอะไรก่อน

อธิบาย step 1...

## Step 2: ทำอะไรต่อ

อธิบาย step 2...

## Notes

- ข้อควรระวัง
- edge cases

C.1 — Frontmatter

FieldRequired?คำอธิบาย
namerequiredชื่อ skill (= ชื่อ folder)
descriptionrequiredบอก Claude ว่า trigger เมื่อไหร่ — สำคัญมาก
argument-hintoptionalแสดงใน autocomplete ว่ารับ args อะไร

description ทำหน้าที่เป็น trigger — Claude ใช้มันตัดสินว่าจะเรียก skill เมื่อไหร่. เขียนให้ครอบคลุม phrases ที่ user จะพิมพ์.

C.2 — $ARGUMENTS

$ARGUMENTS คือ placeholder ที่รับ input จาก user:

/my-skill hello world
# $ARGUMENTS = "hello world"

ใช้ใน SKILL.md body เพื่อบอก Claude ว่า input อยู่ตรงไหน.

D — Pipeline: จาก Ad-hoc สู่ Mature Skill

Skill ไม่ได้เกิดจากการนั่งออกแบบ — มันเกิดจาก การใช้งานซ้ำ:

D.1 — 6 ขั้นตอนวิวัฒนาการ

Step 1: User พิมพ์ /resonance
        ↓
Step 2: /create-shortcut จับได้ (unknown command)
        → infer intent จาก conversation context
        → execute ทันที
        ↓
Step 3: ถาม "Save as /resonance for next time?"
        ↓
Step 4: สร้าง SKILL.md ที่ .claude/skills/resonance/
        ↓
Step 5: Refine — เพิ่ม modes, flags, edge cases
        ↓
Step 6: Distribute ผ่าน npx (optional)

D.2 — /create-shortcut ทำอะไร

/create-shortcut คือ skill factory — สร้าง skill ใหม่จาก terminal:

/create-shortcut                                # list ทั้งหมด
/create-shortcut create deploy "Run tests then deploy"  # สร้างใหม่
/create-shortcut delete deploy                  # ลบ

มันยัง auto-trigger เมื่อ user พิมพ์ command ที่ไม่มี — แทนที่จะ error, มันสร้าง skill ให้เลย.

E — Complexity Spectrum

ระดับตัวอย่างขนาดลักษณะ
Simple /bye ~20 บรรทัด 1 SKILL.md, ทำอย่างเดียว (tmux detach), ไม่มี modes
Medium /forward 271 บรรทัด 1 SKILL.md, หลาย steps, mode flags (--fast), writes files
Complex /rrr 267 + DEEP.md + TEAMMATE.md Main SKILL.md + companion files, subagent modes, hooks

E.1 — /bye (Simple)

ทำแค่ 3 อย่าง: identify BOY name → print farewell → tmux detach-client. ไม่มี flags, ไม่มี modes, ไม่เขียนไฟล์.

E.2 — /rrr (Complex)

มี 5 modes:

/rrr                # Quick retro, main agent only
/rrr --detail       # Full template
/rrr --dig          # Reconstruct past timeline from .jsonl
/rrr --deep         # 5 parallel subagents
/rrr --deep --teammate  # 3 coordinated team agents

Directory structure:

~/.claude/skills/rrr/
├── SKILL.md          ← main (267 lines)
├── DEEP.md           ← instructions สำหรับ --deep mode
└── TEAMMATE.md       ← instructions สำหรับ --teammate mode

SKILL.md หลักบอก Claude ว่า "ถ้า --deep, อ่าน DEEP.md แล้วทำตาม" — conditional include ผ่าน natural language.

F — Key Patterns ที่ Skills ใช้

F.1 — Oracle Root Detection

ทุก skill ที่เขียนไปที่ ψ/ ต้องหา oracle root ก่อน:

# ใน SKILL.md:
## Oracle Root Detection (REQUIRED)
Run: find the repo root that contains ψ/ directory.
Do NOT assume pwd is the oracle repo.

เพราะ Claude อาจรันจาก directory ไหนก็ได้ — ต้องหา root ที่มี ψ/ จริงๆ.

F.2 — Timestamp-First Naming

HHMM_topic.md        ← sort ตาม natural time order
2145_session-retro.md
0830_class-notes.md

F.3 — Mode Flags

/skill              # default (fast, simple)
/skill --fast       # minimal output
/skill --deep       # spawn subagents, thorough
/skill --detail     # full template, more verbose

Convention: default = fast. --deep = expensive (subagents). เสมอ opt-in สำหรับ heavy mode.

F.4 — Description as Trigger

Frontmatter description ทำหน้าที่เป็น trigger condition:

description: 'Use when user says "rrr", "retrospective",
  "wrap up session", "session summary", or at end of work session.'

Claude ใช้ description ตัดสินว่าจะเรียก skill เมื่อไหร่ — เขียนให้ครอบคลุม natural language ที่ user จะพิมพ์.

G — Hooks กับ Skills

Skills ทำงานร่วมกับ settings.json hooks ได้:

Hook Eventเมื่อไหร่ใช้ทำอะไร
PreToolUseก่อน Claude เรียก toolValidate input, gate dangerous commands
PostToolUseหลัง tool เสร็จLog output, trigger follow-up
SessionStartเริ่ม sessionSetup environment, load context
StopClaude หยุดตอบAuto-rrr, cleanup
SessionEndจบ sessionSave state, final cleanup

ตัวอย่าง: fleet มี auto-rrr hook ที่ trigger /rrr เมื่อ session จบ — ไม่ต้องพึ่งว่า BOY จะจำได้เอง.

// settings.json
{
  "hooks": {
    "Stop": [{
      "matcher": "",
      "hooks": [{
        "type": "command",
        "command": "echo 'Run /rrr before ending'"
      }]
    }]
  }
}

Hook + Skill = gate-layer automation. Skill บอก "ทำอะไร", Hook บอก "ทำเมื่อไหร่" — ทั้งคู่ไม่พึ่ง memory.

H — สร้าง Skill ด้วยมือ (Step-by-Step)

Step 1: สร้าง directory

# Project-local
mkdir -p .claude/skills/my-check/

# หรือ Global
mkdir -p ~/.claude/skills/my-check/

Step 2: เขียน SKILL.md

---
name: my-check
description: "Run pre-commit checks. Use when user says 'check', 'pre-commit', or 'verify before commit'."
---

# /my-check

Run lint + test + type-check ก่อน commit.

## Steps

1. Run `bun lint`
2. Run `bun test`
3. Run `bun tsc --noEmit`
4. ถ้าผ่านทั้งหมด → บอก "Ready to commit"
5. ถ้า fail → แสดง errors + suggest fix

Step 3: ทดสอบ

# พิมพ์ใน Claude Code:
/my-check

ถ้า skill ขึ้นใน autocomplete = สำเร็จ.

Step 4: เพิ่ม scripts (optional)

.claude/skills/my-check/
├── SKILL.md
└── scripts/
    └── lint-fix.sh    ← helper script ที่ SKILL.md อ้างถึง

Step 5: เพิ่ม hooks (optional)

ใส่ใน .claude/settings.json ให้ auto-trigger skill:

{
  "hooks": {
    "PreToolUse": [{
      "matcher": "Bash(git commit)",
      "hooks": [{
        "type": "command",
        "command": "echo 'Run /my-check first'"
      }]
    }]
  }
}

J — สรุป

Skill = reusable instruction ที่ Claude ทำตาม. เริ่มจาก ad-hoc slash command → /create-shortcut จับให้ → เก็บเป็น SKILL.md → refine → distribute.

Key takeaways:

  • SKILL.md = markdown instructions, ไม่ใช่ code
  • Frontmatter description = trigger condition — เขียนให้ดี
  • เริ่มเล็ก (20 บรรทัด) แล้วเพิ่ม modes/flags ทีหลัง
  • Hooks + Skills = gate-layer automation — ไม่พึ่ง memory
  • ทุก pattern ที่ทำซ้ำ 3 ครั้ง = ควรเป็น skill
