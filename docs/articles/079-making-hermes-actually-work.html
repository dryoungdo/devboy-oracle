A — บท 1: Tokens & Secrets (ตัวที่ fleet เราต้องอ่าน)

แก่น: Discord bot token ไม่ใช่แค่ string — มันคือ lifecycle object. flow จริง: pass → REST verify → rotation → pm2 env leak → pass-wrapper → safety hooks.

1) Verify ก่อน launch เสมอ (1 วิ vs 10 นาที debug)

Discord มี GET /api/v10/users/@me — Authorization: Bot <token> → 200 = valid, 401 = invalid/revoked. verify ก่อน ไม่งั้น gateway connect ไม่ได้แล้ว log ออกมา cryptic. (เหมือน gate ที่ fleet เราอยากได้ใน discord-wiring)

2) pm2 ~/.pm2/dump.pm2 = the real leak

นี่คือ trap ที่ fleet เราโดนได้ตรงๆ. pm2 save serialize state ของทุก process — รวม env object — ลงไฟล์เดียว. ผลคือ token ของ ~22 apps + ค่า env ทั้งหมด อยู่ใน dump.pm2 เป็น plaintext. file เป็น owner-only แต่ถ้า server โดน compromise = token หลุดทันที.

สาเหตุ: pm2 capture env ตอน pm2 start <script> --env-from-current หรือ inline env: → serialize ลง dump.pm2.

3) Pass-wrapper fix (reusable สำหรับทุก pm2 process ที่ถือ secret)

วิธีถูก: launch wrapper script ที่ source token จาก pass ตอน runtime — หลัง pm2 fork. pm2 เลยเห็นแค่ path ของ script ไม่ใช่ token.

# ~/.hermes/gateway-pm2.sh
#!/usr/bin/env bash
TOK=$(pass show discord/hermes-nous-gateway-token)
export DISCORD_BOT_TOKEN="$TOK"
exec hermes gateway run --replace     # exec (ไม่ใช่ &) → pm2 track PID จริง

# launch
pm2 start ~/.hermes/gateway-pm2.sh --name hermes-gateway
pm2 save

ผลลัพธ์ใน dump.pm2: "script": ".../gateway-pm2.sh", "env": {} — ไม่มี token. กุญแจ: ใช้ exec ไม่ใช่ & — ถ้าใช้ & pm2 จะเห็น script exit ทันที แล้ว restart loop.

4) Token rotation = revoke ทันที → WS Close 4004

พอ reset token ที่ Developer Portal — token เก่าถูก revoke ทันที. gateway ที่รันอยู่ได้ WebSocket close code 4004 (Authentication Failed) แล้ว reconnect ด้วย token เก่าที่ revoke แล้ว → ล้มซ้ำ loop ไม่รู้จบ (adapter.py:679-723 → _set_fatal_error("discord_gateway_task_exited")).

recovery ที่ถูก: pm2 stop → pass insert token ใหม่ → direnv reload (process ต้องอ่าน token ใหม่) → pm2 start. ห้าม pm2 restart / pm2 “Reconnect” — มันใช้ env เก่าที่ capture ไว้.

5) Safety hooks strip secret ออกจาก child env

_SECRET_SUBSTRINGS = (KEY, TOKEN, SECRET, PASSWORD, CREDENTIAL, PASSWD, AUTH) (tests/tools/test_env_passthrough.py:117). env var ที่ชื่อมี substring พวกนี้จะถูก strip ออกจาก child env เว้นมี explicit passthrough — เลยต้อง inject ด้วย $(pass show ...) ตอน shell runtime ไม่ใช่ส่งเป็น command arg.
