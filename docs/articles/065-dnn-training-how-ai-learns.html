DNN Training — AI เรียนรู้ยังไง (เดินลงเขาในหมอก)

บทความ 062 อธิบายว่า ANN "คำนวณ" ยังไง — บทความนี้อธิบายว่ามัน "เรียนรู้" ยังไง = หาค่า weights ที่ดีที่สุดได้อย่างไร

1 — ภาพรวม: จูนกีตาร์พันล้านสาย

ลองนึกภาพ กีตาร์ที่มี 1 พันล้านสาย — สายแต่ละเส้น = weight/parameter ตัวหนึ่ง

  1. ดีดคอร์ด → เสียงแย่มาก (prediction ผิด)
  2. ปรับสายบางเส้น → ดีดอีกที → ดีขึ้นนิดหน่อย
  3. ทำซ้ำเป็นล้านครั้ง → เสียงดีขึ้นเรื่อยๆ
  4. ถึงจุดที่เสียงดี → Training เสร็จ!
ในอุปมาใน AI จริง
"เสียงแย่" = ผิดไปเท่าไหร่Loss Function
"ปรับสาย" = ขยับ weightsGradient Descent
"ทำซ้ำ" = ดีดแล้วปรับอีกEpochs

Claude มี ~100,000 ล้าน parameters ที่ train แบบนี้ — algorithm เดียวกัน แค่ scale ใหญ่มาก

2 — Loss Function: "ผิดไปเท่าไหร่?"

ก่อนจะปรับปรุงอะไรได้ ต้องรู้ก่อนว่า ผิดแค่ไหน

Classification

Model บอก: "มั่นใจ 90% ว่าเป็นแมว"
จริงๆ: เป็นหมา
→ Error สูงมาก!

Regression

Model ทำนาย: ราคาบ้าน 3.5 ล้านบาท
ราคาจริง: 4.0 ล้านบาท
→ Error = 500,000 บาท

เป้าหมาย: ทำให้ total error จาก training data ทั้งหมดต่ำที่สุด

3 — Gradient Descent: เดินลงเขาในหมอก

นี่คือ algorithm หลักของ AI ทุกตัว ลองจินตนาการ:

คุณยืนอยู่บน ภูเขา — หมอกหนามาก มองไม่เห็นอะไรเลย — เป้าหมาย: ลงไปถึง หุบเขา (จุดที่ error ต่ำสุด)

วิธีเดียวที่ทำได้: คลำพื้นใต้เท้า → รู้สึกว่าทางไหนลาดลง → ก้าวไปทางนั้น → ทำซ้ำ

        Error
        ^
   *    |                        * = คุณอยู่ตรงนี้
    \   |
     \  |        เป้าหมาย:
      \ |        เดินลงเนินทีละก้าว
       \|
        \
         \      *  ← ก้าวที่ 2
          \    /
           \  /
            \/  ← หุบเขา (error ต่ำสุด)
   ---------+----------------------> Weights

แต่ละ "ก้าว" = ปรับ weight ทุกตัวนิดหน่อย ไปทิศทางที่ลด error

Learning Rate = ขนาดก้าว

ขนาดก้าวผล
ใหญ่เกินไปกระโดดข้ามหุบเขา กระเด้งไปมา ไม่ลง
เล็กเกินไปค่อยๆ คืบ — train นานเป็นสัปดาห์แทนวัน
พอดีลงเนินสม่ำเสมอ ถึงหุบเขาเร็ว

4 — Backpropagation: ส่งความผิดย้อนกลับ

DNN มีหลาย layer — เมื่อ output ผิด layer ไหนทำผิด?

Backpropagation ตอบคำถามนี้โดย ส่ง error ย้อนกลับทีละ layer

เปรียบเทียบ: งานกลุ่มได้เกรด F

  1. อาจารย์อ่านรายงาน → ระบุส่วนที่แย่
  2. ส่วนนั้นเขียนโดยใคร? → ตามไปหา
  3. คนนั้นใช้ข้อมูลจากใคร? → ตามไปอีก
  4. แต่ละคนรู้ว่าตัวเอง "ผิดแค่ไหน" → ปรับปรุงส่วนของตัวเอง
Input → [Layer 1] → [Layer 2] → [Layer 3] → Output → ERROR!

ส่ง error ย้อนกลับ:
                                                    ← error
                                      ← error ส่วน Layer 3
                        ← error ส่วน Layer 2
          ← error ส่วน Layer 1

แต่ละ layer ปรับ weights ตามส่วนที่ตัวเองรับผิดชอบ

ไม่ใช่เวทมนตร์ — เป็นแคลคูลัส (chain rule) ใช้ทีละ layer แต่ไม่ต้องรู้แคลคูลัสก็เข้าใจหลักการ: ความผิดไหลย้อนกลับ แต่ละ layer แก้ส่วนของตัวเอง

5 — Epochs & Batches

คำความหมายตัวอย่าง
1 EpochModel เห็น training data ครบทั้งหมด 1 รอบดูรูปทั้ง 1 ล้านรูป
Trainingทำ 10-100+ epochsดูรูป 1 ล้านรูป ซ้ำ 50 รอบ
Batchแบ่งข้อมูลเป็นชิ้น ๆ ประมวลผลทีละชิ้นทีละ 256 รูป แทน 1 ล้านรูปพร้อมกัน

6 — เมื่อไหร่ควรหยุด? (Overfitting Trap)

ยิ่ง train นาน training error ยิ่งต่ำ — แต่ถึงจุดหนึ่ง สิ่งอันตรายเกิดขึ้น:

Error
  ^
  |  Training error ──────__________  (ลงเรื่อยๆ)
  |
  |  Test error ──────____/‾‾‾‾‾‾‾‾  (เริ่มขึ้น!)
  |                   ↑
  |              หยุดตรงนี้!
  +──────────────────────────────────> Epochs

Model เริ่ม ท่องจำ training data แทนที่จะเรียนรู้ pattern จริง — เหมือนนักเรียนท่องข้อสอบเก่าคำต่อคำ ทำข้อสอบเก่าได้ 100 คะแนน แต่ข้อสอบใหม่ตก

Early Stopping: ดู test error — พอเริ่มขึ้น = หยุด train ทันที — ตรงนั้นคือ sweet spot

7 — ทำไม "Deep" สำคัญ

Network ตื้น (1-2 layers) = เรียนรู้ pattern ง่ายๆ

Network ลึก (หลายร้อย layers) = เรียนรู้ ลำดับขั้น:

Layer 1-2:    จำ "เส้น" กับ "มุม"
Layer 3-5:    รวม เส้น+มุม เป็น "รูปทรง" (วงกลม, สี่เหลี่ยม)
Layer 6-10:   รวม รูปทรง เป็น "วัตถุ" (ตา, จมูก, ปาก)
Layer 11-20:  รวม วัตถุ เป็น "ใบหน้า"
Layer 21+:    รู้จำว่าเป็น "ใบหน้าของใคร"

AI สมัยนี้ (GPT, Claude) ใช้หลักร้อย layers กับ parameters หลายหมื่นล้าน — algorithm เดียวกันที่อธิบายข้างบนทุกอย่าง แค่ scale ใหญ่กว่า + data เยอะกว่า + GPU หลายพันตัว train เป็นสัปดาห์

คณิตศาสตร์ไม่ใหม่ scale ต่างหากที่เปลี่ยนทุกอย่าง
