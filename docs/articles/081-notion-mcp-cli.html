Notion MCP & CLI — คุม Notion ด้วยเครื่อง ไม่ใช่เมาส์

notion mcp ntn-cli notion-api data-source oauth integration-token fleet-tools
TL;DR. มี 4 ทางเข้า Notion แบบ programmatic — MCP (ให้ AI คุม Notion เป็น tools), ntn CLI (ตัวทางการ เพิ่งเปิด 13 พ.ค. 2026), REST API (curl ได้ทุกอย่าง), และ SDKs (เขียนแอปจริง). ทุกตัววิ่งลง api.notion.com ด้วย token เดียวกันและกฎ data-source เดียวกัน. 2 เรื่องที่พลาดบ่อยสุด: (1) self-hosted MCP ใช้ env NOTION_TOKEN — ไม่ใช่ NOTION_API_KEY; (2) ตั้งแต่ version 2025-09-03 “database” กลายเป็น container ที่ถือ data source — query/สร้าง page ต้องใช้ data_source_id ไม่ใช่ database_id.

1) Hosted — OAuth, ไม่ต้องตั้ง infra

endpoint https://mcp.notion.com/mcp (Streamable HTTP, แนะนำ) มี /sse เป็น fallback. add ใน Claude Code:

claude mcp add --transport http notion https://mcp.notion.com/mcp
# แล้วรัน /mcp ใน Claude Code เพื่อทำ OAuth flow

ข้อจำกัด: access token หมดอายุ ~1 ชม., refresh token หมุน (rotate); ยังอัป file ไม่ได้; และ รัน headless ไม่ได้ เพราะต้องมีคนกด “Allow”.

2) Self-hosted — token, รันที่ไหนก็ได้

claude mcp add notion -e NOTION_TOKEN=ntn_xxxxxxxx -- \
  npx -y @notionhq/notion-mcp-server

หรือใน claude_desktop_config.json / Cursor mcp.json:

{
  "mcpServers": {
    "notion": {
      "command": "npx",
      "args": ["-y", "@notionhq/notion-mcp-server"],
      "env": { "NOTION_TOKEN": "ntn_xxxxxxxx" }
    }
  }
}
กับดัก env var: ตัวแปร auth คือ NOTION_TOKEN — ไม่ใช่ NOTION_API_KEY (อันนั้นเป็นของ server เถื่อนตัวอื่น). form เก่าที่ยังใช้ได้ + เป็น fallback เวลา NOTION_TOKEN ไม่ถูกอ่านในบาง spawned-process คือ JSON header string: OPENAPI_MCP_HEADERS='{"Authorization":"Bearer ntn_xxx","Notion-Version":"2025-09-03"}'.

C — ntn CLI ตัวทางการ (เปิด 13 พ.ค. 2026)

หลายปีที่ ไม่มี Notion CLI ทางการ — เปลี่ยนตอน Notion Developer Platform “3.5” (13 พ.ค. 2026) ที่ปล่อย CLI ชื่อ ntn + Workers (โปรแกรม TypeScript ที่รันใน sandbox ของ Notion).

curl -fsSL https://ntn.dev | bash      # macOS / Linux (แนะนำ)
npm install --global ntn               # ทุก platform
ntn --version

ทำอะไรได้: auth (OAuth, เก็บ credential ใน OS keychain) · ntn api (ยิง API ดิบแบบ authenticated — ไม่ต้อง paste token ใส่ curl) · data sources · upload file · Workers (สร้าง/deploy automation).

framing ตรงๆ: ntn ออกแบบรอบ Workers + auth + raw API passthrough ไม่ใช่ tool “CRUD page จาก bash” แบบครบ. งาน script page/database ทั่วไป — REST API กับ SDK ยังเป็นทางจริง. เช็ค subcommand จริงด้วย ntn --help (per-command flags ยังไม่ครบใน docs ตอน compile). ราคา: CLI ฟรีทุกแพลน; Workers ฟรีช่วง beta แล้วคิดเป็น Notion credits ตั้งแต่ 11 ส.ค. 2026.

D — REST API + การเปลี่ยน “data source” (อ่านอันนี้)

base https://api.notion.com/v1. ทุก request ต้องมี 3 header: bearer token, version, content-type.

# search
curl -s -X POST https://api.notion.com/v1/search \
  -H "Authorization: Bearer $NOTION_TOKEN" \
  -H "Notion-Version: 2026-03-11" \
  -H "Content-Type: application/json" \
  -d '{"query":"Roadmap","filter":{"value":"page","property":"object"}}'

# query data source (ไม่ใช่ /databases/{id}/query แล้ว)
curl -s -X POST https://api.notion.com/v1/data_sources/$DS_ID/query \
  -H "Authorization: Bearer $NOTION_TOKEN" \
  -H "Notion-Version: 2026-03-11" -H "Content-Type: application/json" \
  -d '{"filter":{"property":"Status","select":{"equals":"Done"}}}'

SDK ก็ทางเดียวกัน: npm i @notionhq/client (ทางการ, v5.x, default version 2025-09-03) หรือ pip install notion-client (community — repo ชื่อ notion-sdk-py แต่ package คือ notion-client).

breaking change 2025-09-03 (ตัวที่ทำ script เก่าพังเงียบๆ): “database” กลายเป็น container ที่ถือ data source หลายตัว. “data source” = สิ่งที่เมื่อก่อนเรียก database (ตารางจริง + schema). ของที่เปลี่ยน:
operationเก่าใหม่
query rowsPOST /v1/databases/:id/queryPOST /v1/data_sources/:data_source_id/query
page parent{"database_id":"…"}{"type":"data_source_id","data_source_id":"…"}
search filter"database""data_source"
ทำไมกัดแม้มี data source เดียว: ถ้ามีคนเพิ่ม data source ตัวที่ 2 ในฐานที่ต่อ integration ไว้ — bare database_id จะ กำกวม แล้ว call พัง. ย้ายเลย: เพิ่ม discovery step ดึง data_source_id (GET /v1/databases/{id} → field data_sources[]) แล้วตั้ง header เป็น 2025-09-03+. JS SDK → อัปเป็น v5.
