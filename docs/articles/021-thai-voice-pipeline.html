Thai Voice Pipeline — STT, TTS & Safety Lessons

clubsxai voice stt tts thai safety groq cartesia · 20 พ.ค. 2026 · ❓ raw

STT Evolution (วันเดียว)

No.1 iterate ผ่าน 6 models ภายในวันเดียว — แต่ละ iteration บังคับให้เลือกใหม่ด้วยข้อจำกัดที่ชัดเจน:

  1. SpeechRecognition + Google STT — starting point, network-dependent
  2. faster-whisper tiny — local, เบา แต่ accuracy ต่ำสำหรับภาษาไทย
  3. faster-whisper small — accuracy ดีขึ้น แต่เริ่มกิน CPU
  4. Pathumma medium — Thai-tuned model, ดี แต่ 81°C! CPU sustained high-load ทำให้ bot ไม่เสถียร
  5. faster-whisper tiny (fallback) — ถอยหลังเพราะ thermal
  6. Groq Whisper Large-v3 (final) — cloud inference, เย็น, เร็ว, free tier

Key lesson: local Whisper medium/large = thermal death (80°C+ sustained CPU) สำหรับ persistent bot ที่ต้องทำงานตลอด cloud STT (Groq free tier) ไม่ใช่ premium choice — มันคือ correct default สำหรับ always-on voice bot

Thai STT Challenges

ChallengeปัญหาCurrent Solution
Tonal language 5 tones phonemically contrastive, Whisper trained primarily on formal Thai Groq Whisper Large-v3 handles better than tiny/small
Informal speech Syllable compression — "โน้นึง" ถูกตีความเป็น "โน้ ดึง" Post-processing rules (brittle)
Code-switching Thai-English mid-sentence confuses BPE tokenizer None — known gap
Proper nouns / nicknames "สมโบ" → "สมบัติ" (no training data for Thai nicknames) 30+ trigger name variants (O(N) brittle)

Upgrade path สำหรับ proper noun problem: embedding similarity หรือ phonetic matching แทน hard-coded variant list

Safety Analysis

FeatureStatusGap / Risk
Owner-gate (user_id based) ✅ Implemented Spoofable via soundboard / audio injection
Echo guard (2.5s fixed) ✅ Implemented Should be TTS-duration-aware, not fixed duration
Gibberish filter (>60% short words) ✅ Implemented OK for noise, not for adversarial inputs
Bot ignore list ✅ Implemented Prevents audio loops between bots
Trigger cooldown (8s) ✅ Implemented Anti-spam, adequate
Prompt injection defense ❌ MISSING Highest severity — voice can carry injected instructions to LLM
IPC atomic writes ❌ MISSING Race condition on concurrent triggers
Transcript retention policy ❌ MISSING Privacy risk — no expiry on stored voice transcripts
API spend circuit breaker ❌ MISSING No cap on Groq / Cartesia calls — cost runaway possible

Priority order สำหรับ fix: prompt injection → IPC atomic writes → circuit breaker → transcript policy → echo guard improvement

Groq Free Tier Limits

  • 7,200 audio-seconds/day — ~2 ชั่วโมงของเสียงต่อวัน
  • 100 req/min — เพียงพอสำหรับ voice bot ที่ไม่ได้ตอบพร้อมกันหลายคน
  • Max 25MB/file — ไม่ใช่ปัญหาสำหรับ voice chunks ปกติ
  • ~1,440–3,600 conversational turns/day (assuming 2–5s per turn)

เพียงพอสำหรับ personal bot และ class bot อย่าง DEVBOY-oracle ใน HUMAN SCHOOL — ไม่เพียงพอสำหรับ multi-user production deployment

Key Quote — อาจารย์วิน

"if แกวัดผลจากความประหยัด แต่ in term of คุณภาพ วัดยังไง ใช้กะไม่ใช้ อันไหน หลอน น้อยกว่า"

Quote นี้เกี่ยวกับการเลือก model โดยทั่วไป — แต่ apply ตรงกับ STT selection ด้วย: วัด accuracy ที่ผล (hallucination rate บน Thai informal speech) ไม่ใช่แค่ cost saving local Whisper small อาจประหยัดเงิน แต่ถ้า hallucinate "สมโบ" เป็น "สมบัติ" ทุกครั้ง bot ไม่มีประโยชน์
