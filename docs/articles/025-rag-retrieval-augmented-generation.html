025 — RAG 2026-05-21

RAG — Retrieval-Augmented Generation

rag retrieval hybrid-search arra-oracle vector-search graphrag · 21 พ.ค. 2026 · 🟡 emerging

RAG คืออะไร

Retrieval-Augmented Generation (Lewis et al., 2020) = ดึงข้อมูลที่เกี่ยวข้องจาก knowledge base มาเป็น context ให้ LLM ก่อนตอบ — แก้ปัญหา hallucination + ข้อมูลล้าสมัย

Simple RAG flow: Query → Retrieve → Augment prompt → Generate

ทำไม Context Window ใหญ่ขึ้นแต่ RAG ไม่ตาย

ArgumentReality
"ยัดทุกอย่างใน context ได้แล้ว"Cost ต่อ call สูงมาก — 1M tokens = $3-15/call (Claude/GPT)
"LLM จำได้หมด"Lost-in-the-middle — LLM attention เป็น U-curve, ข้อมูลตรงกลางถูกลืม
"ไม่ต้อง update"Freshness — knowledge cutoff ไม่ช่วยข้อมูลที่เปลี่ยนทุกวัน
"Simple กว่า"RAG ให้ audit trail — รู้ว่าคำตอบมาจากไหน (cite-then-claim)

Hybrid Search = Production Default

ใช้ BM25 (keyword/sparse) + Vector (semantic/dense) ด้วยกัน:

Performance: Hybrid search ให้ +5-15% nDCG เทียบกับ vector-only
Re-ranking เพิ่มอีก +33-40% accuracy (Cohere Rerank, cross-encoder)
MethodStrengthWeakness
BM25 (sparse)Exact keyword match, fast, no GPU neededไม่เข้าใจ synonyms
Vector (dense)Semantic understanding, cross-languageExact keywords อาจพลาด, ต้อง GPU
HybridBest of both worldsComplexity ของ score fusion

Score fusion method ที่ดีที่สุด: Reciprocal Rank Fusion (RRF) — merge rankings จากทั้ง BM25 + vector โดยไม่ต้อง normalize scores

Advanced RAG Patterns

PatternSourceKey Idea
GraphRAG Microsoft, 2024 สร้าง knowledge graph จาก documents → query ทั้ง graph + vector — ดีสำหรับ multi-hop questions
Agentic RAG Various, 2024-25 Agent ตัดสินใจว่าจะ retrieve อะไร เมื่อไหร่ — query planning + iterative retrieval
Self-RAG ICLR 2024 LLM ตัดสินเองว่าต้อง retrieve หรือเปล่า + self-critique คำตอบ
RAPTOR Stanford, 2024 Tree-structured retrieval — สร้าง hierarchical summary clusters
HyDE Gao et al., 2022 Hypothetical Document Embeddings — LLM สร้าง "answer" ก่อน แล้วใช้ embed answer ไป search
CRAG Yan et al., 2024 Corrective RAG — evaluate retrieval quality ก่อนส่งให้ LLM ถ้าแย่ → web search fallback
Contextual Retrieval Anthropic, 2024 เพิ่ม document-level context ให้ทุก chunk ก่อน index → 49% failure reduction
Late Chunking Jina AI, 2024 Embed ทั้ง document ก่อน แล้วค่อย chunk embeddings — preserve cross-chunk context
RAG Fusion Various Multi-query generation + RRF merge — expand query ด้วย LLM-generated variants

Evaluation — RAGAS Framework

RAGAS (Retrieval Augmented Generation Assessment) เป็น standard evaluation framework:

MetricMeasures
Faithfulnessคำตอบ consistent กับ retrieved context หรือเปล่า
Answer Relevancyคำตอบตรงคำถามหรือเปล่า
Context PrecisionRetrieved chunks relevant กับคำถามแค่ไหน
Context Recallดึง relevant info มาครบหรือเปล่า

Known Problems

ProblemImpactMitigation
Lost-in-the-middle LLM attention เป็น U-curve — ข้อมูลตรงกลาง context ถูกลืม จำกัด retrieved chunks, re-rank ให้ relevant อยู่ต้น/ท้าย
Retrieval poisoning >80% flip rate — adversarial chunks เปลี่ยนคำตอบ LLM ได้ Input sanitization, source verification, content integrity checks
Thai word segmentation Tokenizer แย่ → embedding แย่ → retrieval แย่ — cascade failure ใช้ PyThaiNLP/newmm pre-tokenize ก่อน embed
Embedding drift Update embedding model → old vectors ไม่ compatible → re-index ทั้งหมด Version embedding models, store raw text alongside vectors
Rule of thumb: RAG ไม่จำเป็นสำหรับ corpus <50 pages — ยัดใน context ได้เลย ถูกกว่า + ง่ายกว่า

arra-oracle = Minimal RAG System

Fleet v3 มี RAG อยู่แล้ว — arra-oracle ทำ index + search + retrieve:

ComponentCurrentUpgrade Path
StorageSQLite + flat files+ ChromaDB for vectors
SearchKeyword-based+ RRF hybrid scoring (priority 1)
ChunkingDocument-level+ Semantic chunking (priority 2)
FeedbackNone+ Implicit relevance logging (priority 3)
EmbeddingBGE-M3 (correct choice)Keep — best multilingual Thai+EN
GraphNoneGraphRAG when docs >3K
Sovereign RAG: Nearly achievable — local ChromaDB + SQLite + BGE-M3 ทำ RAG ได้ locally ทั้งหมด
เหลือแค่ LLM inference ที่ยัง cloud-dependent (Anthropic API)

DEVBOY Takeaways

  1. Hybrid search (BM25+vector) คือ baseline ที่ต้องมี — arra-oracle ต้องเพิ่ม RRF fusion เป็นอันดับแรก
  2. Contextual Retrieval (Anthropic) ลด failure 49% — เพิ่ม document context ให้ทุก chunk ก่อน index ไม่ยาก
  3. GraphRAG ยังไม่จำเป็น ที่ current arra size (<3K docs) แต่ keep tracking
  4. Thai word segmentation เป็นจุดอ่อนที่ต้อง address — PyThaiNLP pre-tokenize ก่อน embed
  5. Retrieval poisoning เป็น security concern จริง — arra-oracle เป็น internal system (ตัดปัญหา external adversarial content) แต่ถ้าเปิด public ingestion ต้องระวัง
  6. RAG ≠ silver bullet — corpus <50 pages ไม่ต้อง RAG, ยัดใน context ถูกกว่า
