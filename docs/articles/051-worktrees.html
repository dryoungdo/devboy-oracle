A — Git Worktrees คืออะไร

ปกติ 1 repo = 1 branch ที่ checkout อยู่. อยากสลับ branch ต้อง git stash → git checkout → ทำงาน → git checkout กลับ → git stash pop. วุ่นวาย.

Git Worktree แก้ปัญหานี้ — checkout หลาย branch พร้อมกัน แต่ละ branch อยู่ใน directory ของตัวเอง. ทุก worktree share .git object store เดียวกัน — ไม่ต้อง clone ใหม่ ประหยัด disk.

# สร้าง worktree สำหรับ branch fix-auth
git worktree add ../myrepo-fix-auth fix-auth

# ตอนนี้มี 2 directories:
# myrepo/            ← main branch (เดิม)
# myrepo-fix-auth/   ← fix-auth branch (ใหม่)

# ทำงาน 2 อย่างพร้อมกันได้ ไม่ต้อง stash

ข้อจำกัด: 1 branch = 1 worktree เท่านั้น. ถ้า branch main checkout อยู่ใน worktree หนึ่ง จะ checkout ใน worktree อื่นไม่ได้.

B — 4 ชั้นของ Worktree Tooling ใน Fleet

Fleet มีเครื่องมือ worktree 4 ระดับ — จากง่ายไปซับซ้อน:

B.1 — /worktree Skill (Claude Code)

Built-in skill ใน Claude Code. ใช้ EnterWorktree / ExitWorktree tools.

Commandทำอะไร
/worktree fix-authสร้าง worktree ใน .claude/worktrees/fix-auth/, branch worktree-fix-auth
/worktree listแสดง worktrees ที่มีอยู่
/worktree exitกลับ main worktree
/worktree cleanลบ worktrees ที่ไม่ใช้แล้ว

Safety: main branch ไม่มีทางถูกแตะ — ทำงานใน branch แยกเสมอ.

B.2 — maw workon Plugin (maw-js)

Higher-level กว่า /worktree — ครอบทั้ง repo resolution + worktree + tmux + Claude Code:

maw workon glueboy ship-fix
# 1. ghq resolve → ~/ghq/github.com/dryoungdo/glueboy
# 2. สร้าง worktree (branch: agents/1-ship-fix)
# 3. เปิด tmux window ใหม่
# 4. start Claude Code ใน worktree นั้น

2 layouts:

  • Nested (default): repo/agents/<task>/ — worktree อยู่ใน repo
  • Legacy: parent/repo.wt-<task>/ — worktree อยู่ข้าง repo

B.3 — maw oracle-workon (Team Orchestration)

ระดับสูงสุด — compose maw wake --task + maw swarm เข้าด้วยกัน:

maw oracle-workon --task big-refactor --with codex
# 1. สร้าง worktree team
# 2. spawn Claude Code + Codex co-engines
# 3. ทำงาน parallel ใน worktree เดียวกัน

B.4 — maw wake --task

Flag --task ทำให้ maw wake สร้าง worktree เป็นส่วนหนึ่งของ session startup:

maw wake glueboy --task fix-deploy
# wake oracle + สร้าง worktree อัตโนมัติ

maw wake --list
# แสดง active worktrees ทั้งหมด

C — Lifecycle Management

Worktrees ที่สร้างแล้วไม่ลบ = orphan ที่กิน disk + สร้าง confusion. Fleet มี lifecycle management ครบ:

C.1 — Scan & Classify

หา worktree directories (.wt-* และ agents/*) แล้วจำแนก:

Statusเงื่อนไขAction
Activeมี tmux window ที่ยังเปิดอยู่ปล่อยไว้
Staleไม่มี tmux แต่มี uncommitted changesแจ้งเตือน
Orphanไม่มี tmux, ไม่มี changescleanup ได้เลย

C.2 — Cleanup

Cleanup ทำ 4 ขั้นตอน:

  1. Kill tmux window (ถ้ายังเปิดอยู่)
  2. Remove worktree directory (git worktree remove)
  3. Prune git worktree list (git worktree prune)
  4. Delete branch ที่ไม่ใช้แล้ว
# Claude Code skill
/worktree exit --remove

# maw cleanup (fleet-level)
maw done glueboy-ship-fix
# kills tmux + removes worktree + prunes + deletes branch

C.3 — API & Doctor

maw-js มี Elysia REST endpoints สำหรับ programmatic access:

GET  /worktrees           # list ทั้งหมด + status
POST /worktrees/cleanup   # trigger cleanup

team-ops doctor ตรวจหา orphaned worktrees อัตโนมัติ — แจ้งเตือนใน session ถ้าเจอ.

D — ตัวอย่างใช้งานจริง

D.1 — Simple Isolation (1 คน, 1 task)

# กำลังทำงานบน main แล้วต้อง hotfix
/worktree fix-auth

# ทำ fix ใน worktree
git add . && git commit -m "fix: auth token expiry"

# กลับ main
/worktree exit --remove

D.2 — Parallel Oracle Work (Fleet)

# Oracle A ทำ feature
maw workon glueboy ship-fix

# Oracle B ทำ docs (พร้อมกัน, คนละ worktree)
maw workon glueboy update-docs

# เสร็จแล้ว cleanup
maw done glueboy-ship-fix
maw done glueboy-update-docs

D.3 — Team + Co-engine

# Spawn worktree team พร้อม Codex co-engine
maw oracle-workon --task big-refactor --with codex

# Claude ทำ code, Codex ทำ review — parallel ใน worktree เดียวกัน
# เสร็จแล้ว cleanup อัตโนมัติ

E — Gotchas

  1. 1 branch = 1 worktree — พยายาม checkout branch ที่ใช้อยู่แล้ว = error
  2. Orphan worktrees กิน disk — ใช้ maw done หรือ /worktree clean ทุกครั้งที่เสร็จงาน
  3. Shared .git = shared hooks — pre-commit hooks มีผลทุก worktree. ระวัง hook ที่ assume cwd = repo root
  4. Node modules / dependencies — แต่ละ worktree ต้อง npm install แยก (node_modules ไม่ share)
  5. IDE confusion — บาง IDE เปิด worktree แล้วสับสนเรื่อง git state. ใช้ terminal-based workflow ปลอดภัยกว่า
