ARRA Safety Hooks เปลี่ยน "please don't" เป็น "you can't" — Claude Code จะ exit 2 (block) ก่อนที่ command อันตรายจะรัน.

หลักคิด: Oracle Principle 1

"Nothing is Deleted" — ทุกอย่าง append-only. ถ้าต้องลบ ให้ย้ายแทน. ถ้าต้องแก้ history ให้สร้าง commit ใหม่.

Hook นี้บังคับ Principle 1 ที่ tool layer — ไม่ว่า AI จะคิดอะไร command อันตรายจะไม่รัน.

Regex Anchoring — False Positive Prevention

Pattern ทุกตัว anchor ด้วย (^|;|&&|\|\|) เพื่อจับเฉพาะ command จริง ไม่ใช่ text ใน string:

# จับ: rm -rf /tmp/stuff
# จับ: echo ok && rm -rf /tmp
# ไม่จับ: echo "don't use rm -rf"
# ไม่จับ: git commit -m "removed rm -rf usage"

echo "$CMD" | grep -qE '(^|;|&&|\|\|)\s*rm\s+-r?f\s'

Org Membership Check

gh pr create ไม่ได้ block ทุกกรณี — block เฉพาะเมื่อ target org ไม่ใช่ของเรา:

Known Issue: Heredoc False Positive

Issue #1: ถ้าเขียน gh issue create ที่ body มีคำว่า "rm -rf" หรือ "--force" — hook จะ block ทั้ง command. เพราะ regex ดู text ทั้งหมดไม่ได้แยก body ออก.

PR #2 แก้โดยเช็ค first line — ถ้าเป็น gh issue|pr|gist|release create|comment|edit จะข้าม destructive check. แต่ยังจับ chain เช่น gh issue create && rm -rf. มี 38 tests.
