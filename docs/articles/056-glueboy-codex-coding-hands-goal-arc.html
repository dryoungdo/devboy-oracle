B — Codex Coding-Hands Pattern

Captain → GLUEBOY (orchestrator) → codex tile (coding hands). GLUEBOY ไม่ edit source โดยตรง — ส่ง brief ให้ codex ผ่าน maw hey + เฝ้าดูผ่าน maw peek.

B.1 — Spawn pattern

# 1. Create worktree for isolated work
git worktree add ../glueboy-oracle-wt-NN-slug -b feat/NN-slug

# 2. Spawn codex tile pointing at worktree (--dangerously-bypass-approvals-and-sandbox MANDATORY)
maw tile 1 \
  --path "/Users/.../glueboy-oracle-wt-NN-slug" \
  --cmd "codex --dangerously-bypass-approvals-and-sandbox"

# 3. Brief codex via maw hey --force
maw hey 01-glueboy:glueboy-oracle.1 "TASK — ..." --force

# 4. Peek progress
maw peek 01-glueboy:glueboy-oracle.1

# 5. After DONE: kill tile
maw kill 01-glueboy:glueboy-oracle.1

หัวใจของ pattern: --force flag — codex TUI never appears idle (มี Context % indicator), maw's pane-idle detector อ่านเป็น "busy" และ queue ไปที่ inbox แทนที่จะ deliver. --force bypass การตรวจสอบนี้.

B.2 — Bypass-flag เป็น MANDATORY

Default codex profile = readonly (workspace-read). ถ้าไม่มี --dangerously-bypass-approvals-and-sandbox, codex จะสร้าง patch ใน TUI scrollback แต่ ไม่ write files, ไม่ commit, ไม่ push — เป็น "ghost progress" ที่ดูเหมือนทำงานแต่ไม่ได้บันทึก. Cost ~10 min ตอนค้นพบครั้งแรก (2026-05-24).

B.3 — Pane re-indexing trap

หลัง maw kill หรือ maw tile clean, tmux silently re-index panes survivors. .3 อาจกลายเป็น .2, fresh spawns ลงที่ next free index. ที่อยู่ pane ที่จำไว้ก่อน kill อาจชี้ไปที่ codex ตัวอื่น — ALWAYS maw panes หลัง kill เพื่อ re-verify ก่อนส่ง brief ตัวต่อไป.

C — Codex Review Slow-Path Playbook

Codex tile embedded codex review --uncommitted ค้นพบรูปแบบ hang ที่ ~17 นาที no findings emitted ใน 3 ใน 4 codex tiles วันนี้ (#60, #64, #69 second pass). Pattern ชัดเจน: first review pass หา bugs จริงได้ภายใน 5-10 min, second pass drift เข้า scope-creep (อ่าน doctrine text, unrelated files) และค้างที่ ~17 min ไม่มี new findings.

C.1 — Abort sequence (validated 2 ครั้ง)

# 1. Interrupt codex TUI via maw run + escape key
#    (single-keystroke exception ต่อ doctrine "no maw run for AI sessions")
maw run <session>:<pane> $'\x1b'

# 2. Tell codex to kill bg review + ship without re-review
maw hey <session>:<pane> "Codex review hit slow-path (>5 min no findings). \
  Kill bg review via pkill, commit + push + maw pr immediately. \
  Don't restart review." --force

# 3. Codex acknowledges + runs pkill -f 'codex review --uncommitted'
# 4. Codex proceeds to commit + push + maw pr

C.2 — เมื่อไหร่ค่อย abort vs ปล่อย

StateAction
Review emitting findings (เห็นใน peek)ปล่อยให้ทำต่อ — เป็น "value-add" path
Review silent >5 min, no new outputAbort — เป็น "scope-creep" path
Review screen ไม่เปลี่ยน >3 peek cyclesAbort
Total >20 minAbort regardless of findings rate

Cost วันนี้: ~45 min ของ operator-intervention เพื่อ abort 3 hung reviews. แต่ value-add: 3 reviews แรกหา bugs จริง 5+ (รวม private-repo leak fix ที่ codex review จับได้ก่อน post upstream).

C.3 — Mechanical fix (PR #75)

D.2 — The Meta-bug ที่จับตัวเอง

Rule 3 (#57) ค้นพบ false-positive ในวันถัดมา: gate match literal substring "maw tile" ใน git commit message body ที่กล่าวถึง verb. Commit ที่ทำ doctrine fix สำหรับ maw tile clean ถูก block.

# เกิด false-positive:
git commit -m "fix(doctrine): #62 — reference maw tile clean as replacement"
# → SOP-QA GATE [bypass-flag] BLOCKED — Bash tool call starts a codex tile with `maw tile`...
# (แต่ command นี้คือ git commit, ไม่ใช่ tile spawn)

#69 ฟิกซ์โดยเขียน lexical parser (Python script ฝังใน bash) ที่แยกแยะระหว่าง "maw tile ที่ command start" vs "maw tile ใน quoted argument". 30/30 test cases pass (17 original + 13 new false-positive guards).

Override token: GLUEBOY_GATE_BYPASS=<reason> ใน tool input bypass gate + log ไป ~/.claude/.sop-qa-gate/gate.log.
