D — behavior เปลี่ยน — สิ่งที่ทำให้ script พังได้

1. Adaptive thinking เป็นโหมดเดียว. thinking: {"type": "disabled"} ไม่รองรับแล้ว — call ไหน/hook ไหนที่ set ไว้จะ error. คุมความลึกด้วย param effort แทน (low/medium/high — เหมือน xhigh ที่ fleet ตั้ง).
2. Safety reroute. Fable มี 3 classifier — query แนว cybersecurity (exploit/offensive), biology/chemistry ส่วนใหญ่, และ distillation (พยายามดูดโมเดล) จะ fallback ไป Opus 4.8 อัตโนมัติ (ไม่คิดเงิน Fable). Anthropic บอก >95% ของ session ไม่มี fallback. นัยกับ fleet: งาน security ที่สั่ง Fable อาจรันบน Opus 4.8 จริง ไม่ใช่ Fable.
3. gotcha จริง (เจอตอนสลับ glueboy 10 มิ.ย.): /model fable (alias สั้น) ขึ้น “Model 'fable' not found” บน build เก่า แล้วคงเป็น Opus 4.8. แก้: ใช้ full id /model claude-fable-5. ถ้ายัง not found = Claude Code เก่าเกิน (model registry ยังไม่รู้จัก Fable ที่เพิ่งออก 9 มิ.ย.) → claude update แล้วลองใหม่. ถ้ายัง = API key/plan นั้นไม่มี Fable access (เช็ค console).

source: docs.claude.com (adaptive thinking / safety routing) · live debug glueboy 2026-06-10
