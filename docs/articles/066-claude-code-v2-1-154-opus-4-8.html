Source: anthropics/claude-code release v2.1.154 (28 พ.ค. 2026) — ดึงด้วย gh release view ไม่ใช่เดาจากความจำ

วันที่ 28 พ.ค. 2026 Claude Code ออกเวอร์ชัน v2.1.154 — เวอร์ชันที่เปิดยุค Opus 4.8 อย่างเป็นทางการ. บทความนี้สรุปว่ามีอะไรใหม่ ที่ fleet ของเราควรใช้ และ fix ตัวไหนที่กระทบ workflow ของเราตรงๆ

1 — ภาพรวม: เปลี่ยนจาก "ผู้ช่วยคนเดียว" เป็น "หัวหน้าทีม"

ลองนึกภาพช่างคนเดียวที่เก่งมาก — v2.1.154 ไม่ได้แค่ทำให้ช่างคนนั้นเก่งขึ้น (Opus 4.8) แต่ให้เขา "จ้างทีมงานหลักร้อยคน" มาทำงานพร้อมกันใน background ได้ด้วย (dynamic workflows)

3 แกนหลักของเวอร์ชันนี้:

  1. โมเดลฉลาดขึ้น — Opus 4.8 เป็น default
  2. ขยายกำลังได้ — dynamic workflows สั่งงาน agent หลักสิบ-หลักร้อยตัว
  3. เร็วขึ้น+ถูกลง — fast mode บน Opus 4.8

2 — Opus 4.8 (default แล้ว)

โมเดลใหม่สุด Opus 4.8 ตอนนี้เป็น default และ default effort = high

  • งานหนักสุด/ต้องคิดลึก → /effort xhigh (ลึกที่สุด)
  • fleet ของเราตั้ง standard ไว้แล้ว: { "model": "opus", "effortLevel": "xhigh" } ใน .claude/settings.json
  • "model": "opus" = alias ที่ resolve เป็น Opus 4.8 อัตโนมัติ (ไม่ต้อง pin เลขเวอร์ชัน)

เกร็ด: label ของ /effort เปลี่ยนจาก "Speed/Intelligence" เป็น "Faster/Smarter" ให้เข้าใจง่ายขึ้น

3 — Dynamic Workflows (ของใหม่ที่ใหญ่ที่สุด)

เดิม: ถ้าจะให้ agent หลายตัวทำงานพร้อมกัน ต้องตั้งเองทีละตัว. ตอนนี้แค่ "บอก Claude ให้สร้าง workflow" แล้วมัน orchestrate งานข้าม agent หลักสิบถึงหลักร้อยตัวใน background ให้เอง

เปรียบเทียบ:

  • เมื่อก่อน = คุณเป็น foreman ต้องสั่งช่างทีละคน คุมเองหมด
  • ตอนนี้ = คุณบอกเป้าหมาย Claude เป็นคนแจกงาน + คุมทีมหลักร้อยตัวเอง คุณรับแค่ผลลัพธ์

ใช้ /workflows เพื่อดู runs ที่กำลังทำอยู่. เหมาะกับงานใหญ่/ซับซ้อนที่ context เดียวทำไม่ไหว (เช่น migration ทั้ง repo, audit หลายไฟล์, review หลายมุมพร้อมกัน)

หมายเหตุ fleet: นี่คือ engine เดียวกับที่ GLUEBOY ใช้ orchestrate — ตอนนี้เป็น feature มาตรฐานของ Claude Code แล้ว

4 — Fast Mode บน Opus 4.8

โหมดเร็วบน Opus 4.8 ตอนนี้ ราคาถูกลงมาก:

จ่าย  2x  ของ rate มาตรฐาน
ได้   2.5x ของความเร็ว

= คุ้มขึ้นกว่าเดิมเยอะ. เหมาะกับงานที่อยากได้คำตอบเร็ว แต่ยังต้องการความฉลาดระดับ Opus. เปิดด้วย /fast on

5 — ฉลาดขึ้นเรื่อง "เมื่อไหร่ควรถาม"

2 การเปลี่ยนแปลงที่ทำให้คุยลื่นขึ้น:

  • Lean system prompt เป็น default — prompt ระบบสั้นลง (ทุกโมเดลยกเว้น Haiku/Sonnet/Opus 4.7 และเก่ากว่า) = เหลือ context ให้งานจริงมากขึ้น
  • ถาม multiple-choice น้อยลง — Claude จะถามตัวเลือกเฉพาะตอนที่ ตัดสินใจเองไม่ได้จริงๆ ไม่ถามพร่ำเพรื่อเวลาที่มีข้อมูลพอจะทำต่อแล้ว

6 — เครื่องมือ/คำสั่งที่เปลี่ยน

  • /simplify — ตอนนี้เป็น cleanup-only review (reuse / simplify / efficiency / altitude) แล้วแก้ให้เลย — ไม่ใช่ full bug-hunt แบบ /code-review --fix อีกต่อไป
  • claude agents — พิมพ์ ! <command> เพื่อรัน shell command เป็น background session ที่ attach/detach ได้; หรือ claude --bg --exec '<command>'
  • Plugins — ประกาศ defaultEnabled: false ใน plugin.json ได้; เปิดด้วย /plugin หรือ claude plugin enable. แท็บ Discover ปักหมุด plugin ที่ "suggested for this directory"
  • Streaming tool execution เปิดตลอด; stdio MCP subprocess ได้ CLAUDE_CODE_SESSION_ID + CLAUDECODE=1 ใน env

7 — Fix ที่กระทบ fleet ของเราตรงๆ

หลายตัวในนี้คือปัญหาที่เราเคยเจอเอง:

  • bg-session ไม่หลุด goal เวลา /command ยิงแล้ว — เกี่ยวกับ /goal + /loop ของเราโดยตรง (เมื่อก่อน scheduled command ทำให้ลืมเป้าหมาย)
  • worktree-isolation guard — fix subagent ใน bg session ที่แอบเขียนลง shared checkout (เดิมข้าม guard ได้)
  • bg-agent completion notification ไม่ทำให้ขึ้น "out of context" ก่อนเวลาบนโมเดล 1M context
  • pinned bg session ไม่ respawn ทุกนาทีหลัง update แล้ว; bg session ที่ค้างไม่ retire ก็แก้แล้ว
  • ความปลอดภัย: auto-mode detect data-exfiltration (ดูดเนื้อ repo จำนวนมาก) ดีขึ้น; rm -rf $HOME/ (มี trailing slash) ตอนนี้ถูก block แล้ว

8 — สรุป: ใช้อะไรต่อ

  • งานคิดลึก → /effort xhigh บน Opus 4.8
  • งานใหญ่/หลายส่วน → /workflows (dynamic workflows)
  • อยากเร็ว+คุ้ม → /fast on
  • ทำความสะอาดโค้ด → /simplify
  • fleet เราอัพเป็น Opus 4.8 + xhigh หมดแล้ว (settings.json) — floor ที่ต้องมี = Claude Code ≥ 2.1.154

คณิตศาสตร์ของ "ทีมหลักร้อยตัว" ไม่ได้ซับซ้อนขึ้น — แค่ Claude คุมทีมเองได้แล้ว นั่นแหละที่เปลี่ยนเกม
