RTK Benchmark — วัดจริงก่อนเชื่อ

clubsxai rtk benchmark token-savings · 20 พ.ค. 2026 · ❓ raw

โจทย์ทดสอบ

อาจารย์วินสร้างโจทย์ทดสอบ RTK (Rust Token Killer) อย่างจริงจังในห้อง ClubsXai — ไม่เชื่อ marketing claim จนกว่าจะวัดเอง:

  • สร้าง 500-line log file ที่ฝัง ERROR และ FATAL ปนกับ log ปกติ
  • ทดสอบ rtk command เทียบกับ raw cat บน file เดียวกัน
  • วัด token consumption จริงสำหรับทั้งสองแบบ

ผลทดสอบ

TestMethodผล
500-line log (มี ERROR/FATAL)catPassthrough — ไม่มี filtering เลย output ออกมาเต็ม
500-line log (มี ERROR/FATAL)rtkOutput เท่ากับ cat — ไม่ต่าง
Real-world usage (Boom วัดจริง)rtk vs raw4.1% savings บน 800M tokens

ตัวเลข 4.1% ห่างจาก marketing claim 60–90% มาก — แต่นี่คือตัวเลขจริงที่ Boom วัดจากการใช้งานจริงของ fleet

บทเรียน

RTK ไม่ได้ไร้ประโยชน์ — แต่ต้องเข้าใจ scope ที่มันทำงานได้จริง:

  • RTK = hygiene tool ไม่ใช่ silver bullet สำหรับ token savings
  • ประสิทธิภาพขึ้นกับ pattern ของ output ที่ filter ได้ — log file ที่มีข้อมูลสำคัญกระจายทั่วจะ filter ได้น้อย
  • 4.1% ยังมีมูลค่าบน 800M tokens — แต่อย่าตั้งความคาดหวังจาก marketing number
  • Always measure before trust — คำถามอาจารย์วินบังคับ audit assumption ก่อน deploy
อาจารย์วิน: "if แกวัดผลจากความประหยัด แต่ in term of คุณภาพ วัดยังไง ใช้กะไม่ใช้ อันไหน หลอน น้อยกว่า"

คำถามนี้ขยายการถกไปเกินแค่ token count — RTK กรอง context ออกไปบางส่วน ซึ่งอาจทำให้ model "hallucinate น้อยลง" หรือ "เข้าใจบริบทได้แย่ลง" ก็ได้ ขึ้นกับว่า filter ไปถูก token หรือเปล่า คำถามนี้ยังไม่มีคำตอบที่วัดได้จากการทดสอบในห้องเรียน
