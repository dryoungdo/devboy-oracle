2. Agent Architecture (3-Tier Security)

ทุก agent ใช้ architecture เดียวกัน: Orchestrator → Reader (untrusted docs) → Writer. ป้องกัน prompt injection จาก financial documents structurally ไม่ใช่ prompting.
LayerModelToolsRole
OrchestratorOpus 4.7Dispatch onlyไม่แตะ untrusted docs, ไม่เขียนไฟล์
Reader subagentOpus 4.7Read + Grep onlyอ่าน untrusted external docs, return schema-validated JSON only. No MCP, No Write, No Bash
Writer subagentOpus 4.7Write + Editคนเดียวที่เขียนไฟล์ได้. ไม่เปิด external docs โดยตรง
  • ALL 10 agents + ALL subagents ใช้ Opus 4.7 — ไม่มี Sonnet หรือ Haiku ใน managed-agent configs
  • No agent writes to production systems — output staged ใน ./out/ สำหรับ human sign-off
  • Reader output = schema-validated with character-class restrictions — free text ไม่ผ่าน, prompt injection จาก docs propagate ไม่ได้
  • Cross-agent handoffs event-based via handoff_request, routed โดย orchestrate.py (หรือ Temporal/Airflow)

6. Prompt Engineering for Finance

Role-Based Setup

You are assisting a sell-side equity research analyst covering the
software sector. We use a standard DCF framework with a 10-year
projection period and terminal growth rate. All figures should be
in millions USD unless stated otherwise.

Earnings Analysis

Analyze the attached earnings transcript for [Company]. Extract:
(1) reported revenue vs. consensus estimate,
(2) EPS reported vs. consensus,
(3) three most significant changes in forward guidance,
(4) management commentary on margins and cost structure,
(5) any changes to capital allocation priorities.
Flag anything that contradicts the previous quarter's guidance.

DCF Planning (Two-Step)

Create a plan for valuing [Company] (TICKER) using DCF methodology.
List the data you'll need, calculations you'll perform, and
assumptions you'll make. Let me review before you proceed.

Scenario Modeling

Based on the following assumptions, project [Company]'s income
statement for the next three years under three scenarios:
base case (revenue growth X%), bull case (Y%), bear case (Z%).
Hold operating expense growth at [rate] across all scenarios.
Show projected revenue, EBITDA, EBITDA margin, and free cash flow.
Highlight the key sensitivities.

Competitive Financial Comparison

Compare the financial performance of [Company A] and [Company B]
over the last three fiscal years. Create a side-by-side table of
revenue growth, gross margin, operating margin, FCF margin, and
net debt/EBITDA. Identify the three most significant differences
and explain what drives each.

Three-Way Model Verification

I am going to test this scenario: [describe transaction]. Before I
enter the data, tell me exactly what I should expect to see in the
P&L, balance sheet, and cash flow for each affected month. I will
compare your prediction against the actual output. If they do not
match, we will identify where the logic has broken down.

M&A Analysis (from Anthropic's hallucination guide)

As our M&A advisor, analyze this report on the potential acquisition
of AcmeCo by ExampleCorp.

<report>{{REPORT}}</report>

Focus on financial projections, integration risks, and regulatory
hurdles. If you're unsure about any aspect or if the report lacks
necessary information, say "I don't have enough information to
confidently assess this."

7. Anti-Hallucination Techniques (5 Stacking Methods)

Claude still hallucinated historical financial data even with grounding techniques (Wall Street Prep testing, 2026). ALWAYS provide source data. NEVER ask Claude to recall specific financial figures from memory.
#TechniquePrompt Pattern
APermission to say "I don't know""If you're unsure or the report lacks necessary information, say 'I don't have enough information to confidently assess this.'"
BExtract-then-analyze (direct quotes first)"Extract exact quotes most relevant to [topic]. If none found, state 'No relevant quotes found.' Then analyze referencing each quote by number."
CPost-draft citation verification"After drafting, review each claim. Find a direct quote supporting it. If no supporting quote, remove the claim and mark with empty [] brackets."
DExternal knowledge restriction"Use ONLY the data provided. Do not use general knowledge for any financial figures, ratios, or statistics."
EBlind-spot check (CFO Connect)"Identify three most likely failure points. For each: cause, what silent failure looks like, safeguard to add. Then add those safeguards."

13. Limitations & Risks (Brutally Honest)

Failure modes are silent, plausible, and lawsuit-shaped. A hallucinated P/E ratio of 18.3 looks exactly like a real one. No downstream system can distinguish without independent verification.

Hallucination

  • ~3% = benchmark-averaged. In finance: fabricated earnings figures, invented ticker symbols, wrong fiscal year data, blended data from similar-named companies
  • Hallucinations are coherent and plausible, not random noise
  • Precedent: Mata v. Avianca (2023) — ChatGPT invented six court cases. Same failure mode applies to SEC filings, EDGAR references

Numerical Reasoning

  • Claude is NOT a calculator. Token prediction approximates arithmetic
  • Multi-step DCF: accuracy degrades with each step. 10+ operation chains frequently produce wrong answers
  • Large numbers (>8 digits): multiplication unreliable. $1.2B × 1.073 may produce wrong answer
  • Basis points, compound interest, tax-adjusted returns: unreliable without code execution
  • Mitigation: ALWAYS delegate arithmetic to Python/calculator tools

Data Freshness

  • Knowledge cutoff = no current stock prices, interest rates, recent earnings
  • Without live feeds: may provide last price seen in training data as if current
  • 6-month-old bond yield is worse than no data

Security

  • Prompt injection via documents: malicious PDF 10-K could alter Claude's analysis
  • MNPI exposure: prompts containing material non-public information sent to API = insider trading exposure
  • Context window attack: long financial documents may push safety instructions out of effective context

Operational

  • API downtime during market hours = business continuity event
  • Rate limiting spikes during earnings season / Fed announcements
  • Cost: processing 500 10-K filings for screening adds up fast. Budget 10-50x more tokens than prototype testing suggests

15. Quick Start Guide

  1. Choose deployment: Cowork plugins (easiest) vs Managed Agents API (programmatic) vs M365 add-ins (Excel/PowerPoint)
  2. Install: Cowork → Settings > Plugins > paste https://github.com/anthropics/financial-services. หรือ CLI: claude plugin marketplace add anthropics/financial-services
  3. Connect data: Configure MCP connectors (FactSet, internal DBs, CSV/PDF uploads). ต้อง subscription กับ data provider แยก
  4. Use slash commands: /dcf, /comps, /lbo, /earnings, /merger-model, /ic-memo
  5. Apply anti-hallucination: Stack techniques A-E from section 7. Always provide source data.
  6. Validate EVERYTHING: Every number, every citation, every regulatory reference. Human sign-off required.
  7. Audit trail: Implement your own logging layer. Claude API ไม่มี built-in compliance-grade audit
