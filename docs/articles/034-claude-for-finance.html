Part A — Practical Guide

1. Getting Started

Claude for Finance คืออะไร

Claude เป็น AI assistant ที่อ่าน เขียน และวิเคราะห์เอกสารได้. สำหรับสาย finance = เหมือนเพื่อนร่วมงานที่พร้อมช่วย draft memos, อธิบาย accounting standards, สรุป reports ยาว, และช่วยคิด scenarios — ทันที.

Claude ไม่ใช่อะไร

  • ไม่ใช่ Bloomberg terminal — ไม่มี live market data หรือ proprietary databases
  • ไม่ใช่เครื่องคิดเลข — เลขซับซ้อนต้องใช้ code execution (Pro ขึ้นไป)
  • ไม่ใช่ real-time — มี knowledge cutoff, ไม่รู้ราคาหุ้นวันนี้
  • ไม่ใช่ financial advisor — ไม่ให้ investment advice, ห้ามเป็นฐานตัดสินใจเดียว

ถ้าจัดการ sensitive financial data → เริ่มที่ Teams หรือ Enterprise. มี data privacy commitments แข็งกว่า.

5 นาทีแรกหลังสมัคร

  1. Upload เอกสาร non-sensitive (เช่น public earnings report)
  2. ถาม: "Summarize the key risks in this filing"
  3. Follow up: "What questions should I ask management about revenue growth?"
  4. ลอง: "Explain ASC 842 lease accounting like I'm training a new hire"
  5. สังเกต: Claude จำ context การสนทนา — ถามต่อเนื่องได้
Golden Rule: ตรวจเลขเสมอ. Claude เก่งเรื่อง structure, reasoning, ภาษา. แต่เลขไม่ 100%. ดูทุก output เหมือน draft จาก junior analyst — review ก่อนส่งต่อ.

3 ระดับการใช้งาน

  • Tier 1 — Advisor (ความเสี่ยง = ศูนย์): ถามคำถาม ขอคำอธิบาย. ไม่ต้อง upload อะไร
  • Tier 2 — Assistant (ความเสี่ยงต่ำ, verify output): Upload งบ, สัญญา, board deck. ให้ Claude สรุปหรือ flag anomalies. ต้อง review ก่อนใช้
  • Tier 3 — Automation (ต้อง IT setup): API, MCP connectors, agent templates. ต้อง security review

ห้ามใช้ Claude เมื่อ

  • MNPI (Material Non-Public Information) — ห้าม paste insider info เข้า AI tool
  • Real-time trading — Claude ไม่มี live data, ไม่ execute trades
  • Final audit sign-off — Claude ช่วย prepare ได้ แต่ human ต้อง sign
  • Regulatory filings — compliance + legal ต้อง review ก่อน submit

2. Practical Workflows (ใช้ได้เลย)

10 workflows ที่เปิด claude.ai แล้วทำได้ใน 5 นาที. Copy prompt ไปวางได้เลย.

Daily Work

Email Triage (~25 นาที/วัน)

Copy 10-20 emails วาง Claude → ได้ action list เรียงลำดับ + draft replies.

Here are my unread emails. Categorize each as:
URGENT ACTION / REPLY TODAY / FYI ONLY.
For each urgent item, draft a 2-sentence reply.

Meeting Prep (~30 นาที)

วาง agenda + ตัวเลขไตรมาสก่อน → ได้ brief 1 หน้า.

I have a budget review meeting in 1 hour. Here's the
agenda and last quarter's numbers. Give me:
3 key talking points, 2 likely questions I'll be asked,
and suggested answers for each.

Analysis

Earnings Call Breakdown (~45 นาที)

Here's the Q1 earnings call transcript for [Company].
Extract: revenue vs guidance, margin changes, forward
guidance changes, and any red flags management downplayed.
Format as bullet points.

Ratio Analysis (~20 นาที)

Here are income statement and balance sheet for FY2024
and FY2025. Calculate: current ratio, quick ratio,
debt-to-equity, ROE, ROA, gross margin, operating margin,
net margin. Show YoY change and flag anything outside
normal ranges for [industry].

Document Work

Contract Red Flag Scan (~1 ชั่วโมง)

Review this vendor agreement. Flag: auto-renewal clauses,
liability caps, termination penalties, data ownership,
and anything unusual. List each flag with clause number.

Memo Drafting (~40 นาที)

Turn these bullet points into a 1-page finance memo for
the CFO recommending we switch payroll vendors. Tone:
professional, direct. Include cost comparison table and
recommendation paragraph. Bullets: [paste notes]

Excel & Sheets

Formula Generator (~15 นาที)

Write an Excel formula that: looks up employee ID in
column A of Sheet2, returns their department from column C,
but if they started after Jan 1 2025 (date in column D),
append " (New)". Handle errors gracefully.

Variance Commentary (~1 ชั่วโมง)

Here are budget vs actual for Q1 across 12 cost centers.
For any variance exceeding 5%, write a one-sentence
explanation. Format as a table I can paste into PowerPoint.

เวลาที่ประหยัดได้ต่อวัน: ~2-4 ชั่วโมง. เริ่มจาก Email Triage, Ratio Analysis, Formula Generator — ให้ผลใน 2 นาที.

3. Golden Rules: ป้องกัน Hallucination

Claude ยัง hallucinate ตัวเลขการเงินแม้ใช้ grounding techniques (Wall Street Prep testing, 2026). ALWAYS provide source data. NEVER ให้ Claude recall ตัวเลขจาก memory.

5 เทคนิคที่ stack ด้วยกันได้:

  1. Permission to say "I don't know" — เพิ่ม: "If unsure, say 'I don't have enough information'"
  2. Extract-then-analyze — ให้ดึง exact quotes ก่อน แล้วค่อยวิเคราะห์จาก quotes
  3. Post-draft verification — "After drafting, find a supporting quote for each claim. Remove claims without quotes."
  4. Restrict external knowledge — "Use ONLY the data provided. No general knowledge for any figures."
  5. Blind-spot check — "Identify 3 most likely failure points + what silent failure looks like + safeguard."

ทางเลือก 2: DIY Apps Script (ถูกสุด, ควบคุมได้สุด)

ไป Extensions > Apps Script > วาง code นี้ > ตั้ง API key ใน Script Properties:

function CLAUDE(prompt, model) {
  model = model || "claude-sonnet-4-20250514";
  var apiKey = PropertiesService.getScriptProperties()
    .getProperty("ANTHROPIC_API_KEY");

  var payload = {
    "model": model,
    "max_tokens": 1024,
    "messages": [{"role": "user", "content": prompt}]
  };

  var options = {
    "method": "post",
    "headers": {
      "x-api-key": apiKey,
      "anthropic-version": "2023-06-01",
      "content-type": "application/json"
    },
    "payload": JSON.stringify(payload),
    "muteHttpExceptions": true
  };

  var response = UrlFetchApp.fetch(
    "https://api.anthropic.com/v1/messages", options);
  var json = JSON.parse(response.getContentText());
  return json.content[0].text;
}

แล้วใช้ใน cell ได้เลย: =CLAUDE("Categorize: " & A2) แล้วลากลง.

ทางเลือก 3: Copy-Paste (Zero setup)

Copy data จาก Sheets → paste เข้า claude.ai → copy ผลลัพธ์กลับ Sheets. ง่ายสุด ใช้ได้เลย ไม่ต้อง setup อะไร.

6. Agent Architecture

ทุก agent ใช้ 3-tier security architecture เดียวกัน เพื่อป้องกัน prompt injection จาก financial documents:

  • Orchestrator (Opus 4.7) — สั่งงาน, ไม่แตะ untrusted docs, ไม่เขียนไฟล์
  • Reader (Opus 4.7) — อ่าน untrusted docs, return schema-validated JSON only. ไม่มี Write/Bash/MCP
  • Writer (Opus 4.7) — คนเดียวที่เขียนไฟล์ได้. ไม่เปิด external docs โดยตรง
Key: ทุก agent ใช้ Opus 4.7 ทั้งหมด. ไม่มี agent ไหนเขียน production systems — output staged ใน ./out/ สำหรับ human sign-off เสมอ. Reader output ถูก schema-validate จน free text ไม่ผ่าน — prompt injection propagate ไม่ได้.

10. Prompt Engineering

ท่าหลักสำหรับ finance prompting:

Role Setup (ใส่ทุกครั้ง)

You are assisting a sell-side equity research analyst
covering the software sector. Standard DCF, 10-year
projection, terminal growth rate. All figures in millions USD.

Two-Step Analysis (ปลอดภัยกว่า)

Create a plan for valuing [Company] using DCF methodology.
List data needed, calculations, and assumptions.
Let me review before you proceed.

Why: ให้ Claude แสดง plan ก่อน ไม่ใช่กระโดดทำ. ลด hallucination เพราะ review assumptions ได้ก่อน.

Scenario Modeling

Project [Company] income statement for 3 years under:
base (X%), bull (Y%), bear (Z%). Hold opex growth at [rate].
Show revenue, EBITDA, EBITDA margin, FCF per scenario.
Highlight key sensitivities.

Three-Way Verification (CFO Connect)

I will test this scenario: [describe]. Before I enter data,
tell me exactly what to expect in P&L, balance sheet, and
cash flow for each affected month. I will compare against
actual output. If mismatch, identify where logic broke.

Why: ให้ Claude ทำนายก่อน แล้วเราเทียบ. จับ logic errors ได้ก่อนเข้า production model.

13. Compliance & Regulatory

Certifications

SOC 2 Type II + ISO 27001 + ISO 42001 (AI management). HIPAA BAA: Enterprise only. FedRAMP: via AWS Bedrock.

Data Handling

  • Zero Data Retention: Available on API. Anthropic ไม่เก็บ prompts/completions
  • Data residency: US เป็นหลัก. EU: ผ่าน Bedrock/Vertex EU regions
  • Audit trail: ต้อง implement logging layer เอง — Claude API ไม่มี built-in compliance-grade audit

Regulatory Landscape

  • SEC: AI outputs subject to existing supervisory obligations. Advisor bears liability.
  • FINRA 2026: Dedicated GenAI section. Firms must demonstrate supervisory controls.
  • SR 11-7: LLM used for financial analysis IS a model under SR 11-7. Requires validation + monitoring. Most firms not treating it as such.
  • EU AI Act: Financial credit scoring = high-risk (Annex III). Conformity assessments required.
  • Thailand BOT: FEAT principles (Fairness, Ethics, Accountability, Transparency). Principles-based. Key: explainability of AI-driven credit/insurance decisions.

15. Limitations & Risks

Failure modes are silent, plausible, and lawsuit-shaped. Hallucinated P/E of 18.3 looks identical to a real one.
  • Hallucination: ~3% benchmark-averaged. ในทาง finance: fabricated earnings, invented tickers, wrong fiscal year. Coherent and plausible, not random noise
  • ไม่ใช่เครื่องคิดเลข: Multi-step DCF accuracy degrades with each step. Large numbers (>8 digits) multiplication unreliable. Always delegate math to code execution
  • Data freshness: Knowledge cutoff = ไม่รู้ราคาวันนี้. อาจให้ราคาเก่าเหมือนเป็นปัจจุบัน
  • Prompt injection: Malicious PDF 10-K could alter analysis. MNPI in prompts = insider trading exposure
  • Operational: API downtime during market hours. Rate limiting during earnings season. Cost: budget 10-50x more than prototype testing
