Part A — สรุปที่ต้องรู้

มันคืออะไร?

omnivoice-thai คือ Thai fine-tune ของ k2-fsa/OmniVoice — โมเดล TTS (Text-to-Speech) รุ่นใหม่จากทีม Daniel Povey (คนสร้าง Kaldi — ตำนานวงการ speech).

  • ทำได้: แปลงข้อความเป็นเสียงพูดไทย, clone เสียงจากตัวอย่างสั้นๆ, ออกแบบเสียง (เลือกเพศ/pitch/accent)
  • Open-source: Apache 2.0 — ใช้ commercial ได้เลย
  • Self-host ได้: รันบนเครื่องตัวเอง ไม่ต้องพึ่ง API ใคร
  • สถานะ: proof-of-concept — ยังไม่ production-ready

ลองใช้ยังไง?

pip install omnivoice

from omnivoice import OmniVoice
import soundfile as sf

model = OmniVoice.from_pretrained("hotdogs/omnivoice-thai")

# TTS ปกติ
audio = model.generate(
    text="สวัสดีครับ วันนี้อากาศดีมากเลย",
    instruct="male, low pitch",
)
sf.write("output.wav", audio[0], 24000)

# Voice cloning — ใช้เสียงตัวอย่าง
audio = model.generate(
    text="ยินดีต้อนรับเข้าสู่ระบบ",
    prompt_speech="reference_voice.wav",
)
sf.write("cloned.wav", audio[0], 24000)

มี Colab notebook ใน HuggingFace repo — กด Run ได้เลยไม่ต้อง setup เอง.

ข้อควรระวัง: โมเดลหนัก ~7.4 GB, ต้องการ GPU (แนะนำ VRAM 8GB+). CPU ช้ามาก.

Voice Design — สั่งด้วยข้อความ

สิ่งที่ OmniVoice ทำได้แต่ Cartesia/edge-tts ทำไม่ได้ — สั่ง instruct เป็นภาษาอังกฤษ:

# เสียงผู้หญิง pitch สูง
model.generate(text="...", instruct="female, high pitch, young")

# เสียงผู้ชายแก่ พูดช้า
model.generate(text="...", instruct="male, low pitch, elderly, slow")

# ใส่เสียงหัวเราะ
model.generate(text="นี่มัน [laughter] ตลกมาก")

สถานะ — ใช้ production ได้ยัง?

ยังไม่ได้. เหตุผล:

  • Train ด้วย data แค่ 12.6 ชม. / 2 speakers — น้อยมากสำหรับ TTS
  • ไม่มี evaluation metrics (MOS, WER, speaker similarity)
  • Base model เน้น English/Chinese — Thai เป็น "new territory"
  • Loss ยัง fluctuate (smoothed ~4.4)

แต่ base model แข็งมาก (581K ชม., 600+ ภาษา, 2.1M+ downloads). ถ้ามีคนมา fine-tune ด้วย data ไทยเยอะกว่านี้ คุณภาพจะดีขึ้นมาก.

Capabilities ทั้งหมด

  1. Text-to-Speech — ข้อความ → เสียงพูด (Thai + English)
  2. Zero-shot Voice Cloning — clone เสียงจาก audio สั้นๆ ไม่ต้อง train เพิ่ม
  3. Voice Design — สั่งลักษณะเสียงด้วยข้อความ (gender, pitch, accent, age)
  4. Non-verbal Symbols — [laughter], pauses
  5. Pronunciation Control — กำกับเสียงอ่านด้วย pinyin/phonemes

Limitations & Risks

  • Data น้อยมาก — 12.6 ชม. จาก 2 speakers ไม่พอสำหรับ TTS ที่ดี (เทียบ: Cartesia train ด้วย data หลายพันชั่วโมง)
  • ไม่มี evaluation — ไม่มี MOS score, WER, speaker similarity ที่วัดจริง
  • Transcript อาจผิด — dataset ใช้ ASR-generated transcripts (confidence 0.78–1.0)
  • Training ไม่จบ — 1,747/30,000 steps (auto-stop เมื่อ loss < 3.0)
  • GPU required — ต้องมี VRAM 8GB+ สำหรับ inference

โอกาสที่น่าจับตา

  1. Fine-tune ด้วย data เยอะกว่า — GigaSpeech2 มี Thai data อีกมาก. ถ้าใครมา train ด้วย 100+ ชม. คุณภาพจะต่างชัด.
  2. omnivoice.cpp — C++ port รันบน CPU ได้. ถ้า Thai fine-tune ออกเป็น GGUF format จะรันบน Mac/PC ไม่ต้องมี GPU.
  3. Voice cloning สำหรับ fleet — ปัจจุบัน fleet ใช้ fixed voices (Suda/Somchai). OmniVoice เปิดโอกาส custom voice per BOY.
  4. Self-hosted privacy — ไม่ต้องส่งเสียงไปยัง Cartesia/Microsoft. รันในเครื่องเอง.
