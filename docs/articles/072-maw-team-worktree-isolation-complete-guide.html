Thesis (SomBo capstone): “A team is a file; two idempotent verbs bind that file to live panes; every operation is a non-destructive move.”

1 — mental model: a team is a file (3 ชั้น แต่ truth แค่ 2)

3 — config hierarchy + engine resolution

discovery regex: maw\.config\.(\d+)(\.local)?\.json. merge ascending (load.ts:119) → weight สูง = apply ทีหลัง = ชนะ:

built-in → 50 (user ~/.config/maw) → 80 (project .maw) → 80.local (wins)
  • restricted keys (federationToken / peers / port / bind / sessions) → ต้องอยู่ .50.json หรือ .local.json เท่านั้น (maw warns ถ้าอยู่ project scope). .maw/*.local.json ต้อง gitignore.
  • resolution (command-logic.ts:117): commands[engine] ?? commands.default ?? "claude". engine map = source of truth ไม่ใช่ binary — -e codex ถ้าไม่มี key = silent no-op ตกไป default.
  • channel ≠ engine: channel = Discord/Telegram/Slack bridge (append --channels), คนละชั้นกับ engine.

4 — pipeline: plan → preflight → load → spawn

stageทำอะไร
maw team plan <charter>read-only — list artifacts, ไม่เขียน
maw team preflight <charter>6 checks (name, unique roles, no collision, targets, governance)
maw team load <charter> --no-spawnmaterialize live config + inboxes + manifest — ไม่ spawn pane (--no-spawn required)
spawnแยกขั้น — ผ่าน maw wake --wt (charter flow v2.0.1 ไม่ spawn pane เอง)

5 — keystone spawn: maw wake --wt

maw wake <Org>/<repo> --wt <slot> -e <engine> --session <sess>
  • ทำ worktree + branch + pane + engine ในคำสั่งเดียว (แทน manual git worktree add + maw new --split --path)
  • org-qualified name required — bare name resolve ผิด repo เงียบๆ
  • --wt สร้าง/reuse worktree (layout: repo/agents/N-slot); --fresh = slot ใหม่สะอาด; --list = preview ไม่สร้าง; --session = ยิงเข้า fleet session ที่มีอยู่

6 — worktree isolation (กับดักอันดับ 1)

writer แต่ละตัวต้องมี worktree ของตัวเอง ไม่งั้นเขียนชนกัน โดยไม่มี conflict marker — แค่งานหาย. maw wake --wt = isolated (writers); maw swarm = shared cwd (read/review เท่านั้น).

พิสูจน์ isolation ด้วย pid→cwd ไม่ใช่ roster (จาก #071 ที่ DEVBOY รันจริง): 
coder-1  pid 2603473  cwd .../agents/1-coder-1  [agents/1-coder-1]
coder-2  pid 2603654  cwd .../agents/1-coder-2  [agents/1-coder-2]
→ readlink /proc/$pid/cwd ต่างกัน = isolated จริง

7 — idempotent verbs: /team-up + /gather

  • /team-up — idempotent ensure: live→skip, dead→relaunch, missing→wake (--gather compose ด้วย). re-runnable ปลอดภัย
  • /gather — join-pane เอา member มาข้าง lead (--scatter กลับ). join-pane = MOVE (process รันต่อ) ไม่ใช่ copy
  • /local-team — manual baseline (declare→spawn→prove→teardown)

6-step flow ที่ P'Nat สอน: Declare → Validate → Spawn → Ensure → Gather → Drive.

9 — liveness: pane_current_command ไม่ใช่ dot

เช็คว่า pane เป็นๆ ไหม ดู pane_current_command — zsh/bash/sh = ตายแล้ว (drop เป็น shell). ไม่ใช่ ●/◌ status (quiet-live agent ทำให้ stale ได้). เช่น Hermes daemon = python ไม่ใช่ claude → naive check จะนึกว่าตาย.

“prove by cwd / pane_current_command — the roster can lie”

10 — gotchas (รวมจาก proof + masterclass)

  • engine resolution version-gated — บน maw v26.5.21 maw wake -e codex ไม่อ่าน project .maw/maw.config.80.json → ตก global default → ถ้า global = claude --continue + ไม่มี conv → dead bash (DEVBOY เจอเองใน #071). workaround: ตั้ง commands ใน global, หรือ upgrade maw
  • --split same-session พัง (#1835 nested attach) → coder กลายเป็น separate window แทน split
  • YAML > folded scalar parser ไม่กิน → ใช้ single-line quoted string
  • claude --continue ไม่มี conversation เดิม → "No conversation found" → dead shell
  • ใช้ python3 สำหรับ JSON ไม่ใช่ jq (ไม่ guaranteed ทุกเครื่อง)
  • tmux session+window ชื่อซ้ำ glob-collide → ตั้งชื่อต่างกัน
  • maw config show / maw promote = verb ใหม่ ไม่มีบน v26.5.21 (version gap)

11 — เลือกอะไร: team charter vs team-tile vs subagent

ใช้เมื่อ
team charter + maw wake --wt (บทนี้)writer หลายตัวเขียน parallel ใน worktree แยก, repo register แล้ว, ต้องการ lifecycle + respawn
team-tile (#031)spawn codex tiles ใน pane (50/50 กับ lead), งานชั่วคราว, ไม่ต้อง worktree lifecycle
subagent (Agent tool)งานในรอบเดียว ไม่ต้อง process แยก/survive session
