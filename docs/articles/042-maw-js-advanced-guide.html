Part A — สรุปที่ต้องรู้

maw คืออะไร?

maw คือ multi-oracle session manager — สร้างบน tmux + HTTP federation. จัดการ lifecycle ของ Oracle (AI agent) ข้ามหลายเครื่อง, ส่งข้อความระหว่าง agents, และ orchestrate multi-agent teams.

Part B — Deep Dive

1. Lifecycle Commands

maw wake <oracle>

Primary way to start. Fuzzy-match name, auto-clone จาก GitHub ถ้าจำเป็น, สร้าง tmux session.

maw wake glueboy              # wake by name
maw wake glueboy --split      # wake ใน split pane
maw wake glueboy --model opus # override model
maw wake --all                # wake ทุก oracle

maw kill <target>

maw kill glueboy              # kill pane
maw kill 01-devboy            # kill by session name
maw kill --all                # kill ทุกอัน (ระวัง!)

maw new <name>

สร้าง tmux session เปล่า (ไม่มี oracle) สำหรับ shell tasks.

maw scaffold <name>

สร้าง repo skeleton (CLAUDE.md, ψ/, .claude/) โดยไม่ start อะไร.

2. Communication Commands

maw hey <target> 'msg' — Primary IPC

คำสั่งที่สำคัญที่สุด. ส่งข้อความระหว่าง oracles ข้ามเครื่อง ผ่าน federation. เห็นใน maw web UI.

# Cross-node (ใช้บ่อยสุด)
maw hey mac-studio:11-glueboy:glueboy-oracle 'update [clinic-drdo:devboy]'

# Local
maw hey devboy 'quick question'

# Format: <node>:<session>:<pane> หรือ <agent-name>

สำคัญ:

  • ต้อง mutual peer handshake (maw peers add ทั้ง 2 ฝั่ง)
  • ลงท้ายด้วย [node:handle] เสมอ เพื่อบอกว่าใครส่ง
  • หลังส่ง ตรวจด้วย maw peek ภายใน 3-5 วินาที

maw send <target> <text>

Low-level tmux typing. ไม่มี federation, ไม่มี auth. ใช้ local automation เท่านั้น.

maw broadcast 'msg'

ส่งทุก agent. กว้างเกินสำหรับ 1:1 — ใช้ maw hey แทน.

3. View & Inspection

maw ls

maw ls                # compact
maw ls -v             # verbose (pids, memory, uptime)
maw ls -r             # recent (sorted by activity)
maw ls -a             # roster (ทุกตัว, run หรือไม่ run)
maw ls --json         # machine-readable

maw peek <target>

maw peek devboy                                    # local
maw peek mac-studio:11-glueboy:glueboy-oracle      # remote
maw peek devboy --lines 50                          # last 50 lines

Pattern: หลัง maw hey ให้ maw peek เสมอ 3-5 วิ เพื่อ verify delivery.

maw panes

List ทุก panes ทุก sessions.

5. Federation

Architecture — 2-Layer Auth

Federation auth มี 2 layers ที่ต้อง pass ทั้งคู่:

LayerMechanismSource
1. HMACSHA-256 sign body + timestampfederationToken ใน config
2. ed25519Per-peer signature on "from" field~/.maw/peers.json

Setup (New Node)

# บน NEW node:
# 1. แก้ ~/.config/maw/maw.config.json — เพิ่ม namedPeers
# 2. ตั้ง federationToken (ต้องตรงกับ fleet)
# 3. Add peer
maw peers add <existing-node>

# บน EXISTING node (mutual!):
maw peers add <new-node>

# Verify
maw federation status --verify
maw hey <new-node>:<agent> 'test [my-node:my-handle]'

Federation Commands

maw federation status           # peer health
maw federation status --verify  # health check ทุก pair
maw federation sync             # sync peer state
maw federation sync --dry-run   # preview
maw federation sync --prune     # ลบ stale entries
maw discover                    # inventory ทั้งหมด
maw discover --tree             # hierarchical view

Message Delivery Flow

Local: CLI → resolve agent → tmux send-keys → oracle receives

Cross-node: CLI → resolve agent → lookup namedPeers URL → HTTP POST /api/send → HMAC sign → ed25519 sign → remote maw verifies → tmux send-keys → oracle receives

Part E — Decision Trees & Workflows

Decision Tree: "ฉันอยาก..."

เปิด Oracle

  • Oracle ใหม่ (ยังไม่เคย init) → maw scaffold <name> + maw wake
  • Oracle เดิม, session ใหม่ → maw wake <name>
  • ทำงานคู่กัน → maw bring <name>
  • Wake ทั้งหมด → maw wake --all
  • Shell เปล่า → maw new <name>

ส่งข้อความ

  • ข้ามเครื่อง → maw hey <node>:<session>:<pane> 'msg'
  • เครื่องเดียวกัน → maw hey <agent> 'msg'
  • ทั้ง fleet → maw broadcast 'msg'
  • Verify delivery → maw peek <target> (3-5 วิหลังส่ง)

ดูสถานะ

  • Sessions ที่ run → maw ls
  • Oracle ที่ลงทะเบียน → maw oracle ls
  • Oracle เฉพาะตัว → maw peek <target>
  • Federation connectivity → maw federation status
  • Full inventory → maw discover

แก้ปัญหา

  • 401 auth failure → pm2 logs maw --err → ดู error type
  • Oracle ไม่ตอบ → maw peek <target>
  • Zombie panes → maw cleanup
  • Config ผิด → maw fleet validate + maw fleet doctor --fix
  • หลัง crash → maw fleet restore --all
