Part A — สรุปที่ต้องรู้

มันคืออะไร?

maw-rs คือ Rust rewrite ของ maw-js — CLI ที่จัดการ multi-agent fleet ผ่าน tmux. ใช้บริหาร Oracle agents ทั้งหมด: ตั้งชื่อ session, federation auth, plugin management, target routing, peer discovery.

  • Version: v0.1.0-alpha.1
  • License: BUSL-1.1
  • Author: Nat (Soul-Brews-Studio)
  • สถานะ: Phase 1 done (pure logic) — ยังไม่รัน production

3 Phases — Roadmap

  1. Phase 1 (done) — Port pure/deterministic logic. 21 crates, 95.53% coverage. JSON fixture parity กับ maw-js.
  2. Phase 2 (in progress) — Wire runtime IO: tmux process adapter, HTTP federation, Zenoh transport. ใช้ fake-backed adapters ก่อน real IO.
  3. Phase 3 (future) — clap CLI binary. Port ls, hey, peek ทีละ command. maw-js + maw-rs run side-by-side จนกว่าจะ proven.

Design Principles

  • Same-test-first — Rust behavior ยอมรับได้ก็ต่อเมื่อ match maw-js fixture/golden output
  • Fake before IO — tmux, filesystem, HTTP ใช้ test doubles ก่อน real side effects
  • unsafe = forbid — ห้าม unsafe code ทั้ง workspace
  • Stabilize before expand — จบ crate ปัจจุบันก่อนเปิด lane ใหม่

Fixture Parity Testing

ระบบทดสอบ key ของ maw-rs — JSON fixtures คัดลอกจาก maw-js/test/spec/ มาใช้ตรงๆ:

// maw-rs test loads same JSON as maw-js test
#[test]
fn resolve_target_fixtures() {
    let fixtures: Vec<Fixture> = serde_json::from_str(
        include_str!("fixtures/resolve-target.fixtures.json")
    ).unwrap();
    for f in fixtures {
        assert_eq!(resolve_target(&f.input), f.expected);
    }
}

ทั้ง 2 engines ต้อง agree — ถ้า Rust output ไม่ตรง maw-js = test fail.

Key Rust Patterns

  • #[must_use] บน query functions ทุกตัว
  • BTreeMap/BTreeSet แทน HashMap — deterministic output
  • Newtype wrapper Headers(BTreeMap) with case-folding on insert
  • TmuxRunner trait injection — test ไม่ต้องรัน tmux จริง
  • CliOutput { code, stdout, stderr } — every command testable in-process
  • OnceLock<Mutex<...>> สำหรับ plugin registry static state

Fleet Relevance

ทำไมสำคัญ: maw คือ backbone ของ fleet ทั้งหมด. ถ้า maw-rs สำเร็จ:

  • Startup <5ms แทน ~200ms — maw hey จะเร็วขึ้น 40x
  • Single binary ไม่ต้อง Bun/Node runtime
  • WASM plugins เปิดโอกาสให้เขียน plugin เป็น Rust compile ลง WASM
  • Memory-safe federation auth ไม่ต้องห่วง runtime vulnerabilities
