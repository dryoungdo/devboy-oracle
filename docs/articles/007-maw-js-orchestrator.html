maw-js — Multi-Agent Workflow Orchestrator

maw orchestrator bun tmux · 20 พ.ค. 2026 · ❓ raw

maw-js คืออะไร

maw-js (Multi-Agent Workflow) คือ orchestrator สำหรับจัดการ AI agent หลายตัวพร้อมกัน ผ่าน tmux + CLI ทำงานบน Bun 1.3+ เขียนด้วย TypeScript แบบ monorepo

จุดเด่นคือสั่ง agent ข้ามเครื่องได้ผ่าน federation — ใช้ Zenoh messaging + HMAC-SHA256 signing

Core Features

  • CLI-driven orchestration — wake agents ใน tmux, ส่ง task, monitor, ดู cost จาก terminal เดียว
  • Cross-machine federation — deploy บน 1 node หรือ 20 node ด้วย command เดียวกัน
  • Web UI — React + Three.js federation lens (install แยกจาก maw-ui repo)
  • Agent spawning — maw bud สร้าง oracle, maw wake จาก repo URL หรือ issue prompt
  • Plugin system — extensible ผ่าน maw-plugin-registry

คำสั่งที่ใช้บ่อย

# จัดการ agent
maw ls                    # list ทุก agent
maw ls --active           # เฉพาะที่ active ล่าสุด
maw wake <oracle>         # start agent ใน tmux
maw a <oracle>            # attach เข้าไปดู
maw bring <oracle>        # shorthand for wake --split
maw stop <oracle>         # หยุด agent

# ส่งงาน
maw hey <oracle> "prompt" # ส่ง prompt ไปที่ agent
maw peek <oracle>         # ดู pane output
maw capture <oracle> -n N # capture N บรรทัดล่าสุด

# Federation
maw federation status     # ดู peers
maw osmosis --push <host> # sync sessions ข้ามเครื่อง

Architecture

maw-js ทำงานเป็น monorepo:

  • src/cli.ts — CLI entry point + binary distribution
  • Server — WebSocket/HTTP API on port :3456
  • UI — React/Three.js bundle (install via maw ui install)
  • Plugin transport — hub-based routing, manifest lock/install
  • Federation — Zenoh-based cross-machine messaging (@eclipse-zenoh/zenoh-ts)
