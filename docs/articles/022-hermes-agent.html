Hermes Agent — Self-Improving AI Agent Platform

hermes nousresearch python multi-provider self-improving mcp · 20 พ.ค. 2026 · ❓ raw

Hermes Agent คืออะไร

Hermes Agent คือ open-source AI agent platform จาก Nous Research — Python 3.11+, MIT license, v0.14.0

จุดเด่นหลัก: self-improving — agent สร้างและปรับปรุง skills ของตัวเองอัตโนมัติ + deploy ได้ทุกที่ (VPS, GPU cluster, serverless) ไม่ผูกกับ laptop

Architecture — 3 Entry Points

Entry PointFilePurpose
CLI hermes_cli/main.py Interactive terminal + slash-command autocomplete
Gateway gateway/run.py Messaging platform dispatcher — 22 platforms
ACP Adapter acp_adapter/server.py Editor integration (VS Code, Zed, JetBrains)

Core loop อยู่ใน AIAgent class (run_agent.py, ~2,000 lines) — prompt building, provider resolution, tool dispatch, session persistence

Self-Improving Skills

นี่คือ feature ที่แยก Hermes จาก agent อื่น — agent สร้าง skill ใหม่ได้เอง:

  1. หลังจาก task ที่ซับซ้อน agent เขียน SKILL.md อัตโนมัติลงใน ~/.hermes/skills/
  2. Skills มี progressive disclosure (Level 0: metadata, Level 1: full content, Level 2: references)
  3. ใช้งานครั้งต่อไป agent ปรับปรุง SKILL.md ที่มีอยู่ — self-improvement loop
  4. Compatible กับ agentskills.io open standard
  5. Security gate: skills_guard.py scan agent-created skills (opt-in)

22 Messaging Platforms

Gateway dispatcher รองรับ 22 platforms ผ่าน BasePlatformAdapter:

  • Chat: Telegram, Discord, Slack, Matrix, Mattermost, IRC
  • Mobile: WhatsApp, Signal, LINE, WeChat, DingTalk, Feishu
  • Email/SMS: Email adapter, SMS adapter
  • IoT: HomeAssistant, BlueBubbles
  • Others: SimpleX Chat, WeChat Work, Yuanbao

ทุก platform รองรับ voice memo transcription + cross-platform conversation continuity + cron job delivery

29 LLM Providers

Plugin architecture — แต่ละ provider เป็น directory ใน plugins/model-providers/ มี plugin.yaml:

  • Cloud: Anthropic, OpenAI, Google Gemini, xAI Grok, DeepSeek, Qwen
  • Enterprise: AWS Bedrock, Azure Foundry, NVIDIA NIM
  • Aggregator: OpenRouter (200+ models), NovitaAI
  • Local: Ollama, custom endpoints
  • Proxy: hermes proxy bridges OAuth-authed providers (Claude Pro, ChatGPT Pro, SuperGrok) เป็น OpenAI-compatible endpoint

MCP + OAuth + ACP

StandardRoleImplementation
MCP (consumer) เรียกใช้ external tools tools/mcp_tool.py
MCP (server) Expose conversations ให้ Claude Desktop, Cursor hermes mcp serve (stdio + Streamable HTTP)
MCP OAuth Auth สำหรับ OAuth-protected MCP servers tools/mcp_oauth_manager.py
ACP Editor integration hermes acp — VS Code, Zed, JetBrains

Trajectory Compression

Feature สำหรับ research — แปลง agent session เป็น training data:

  1. Protect first turns (system prompt, initial request, first response)
  2. Protect last N turns (final conclusions)
  3. Compress MIDDLE turns — replace ด้วย summary message
  4. Target budget: 16K tokens (configurable)

ใช้กับ batch_runner.py + mini_swe_runner.py สำหรับสร้าง training data ให้ tool-calling models รุ่นต่อไป

Memory System

  • MEMORY.md (2,200 chars) — agent notes, persistent across sessions
  • USER.md (1,375 chars) — user profile
  • Frozen injection เข้า system prompt ตอน session start (preserve prefix cache)
  • External backends: Honcho, Mem0, Hindsight, RetainDB, ByteRover
  • FTS5 search across past sessions + LLM summarization (/insights command)

Scale & Testing

  • 70+ tools across 28 toolsets
  • 1,137 test files covering agent loop, tools, providers, compression, platforms
  • Core: run_agent.py (~2,000 lines), hermes_state.py (SQLite+FTS5)
  • pytest + pytest-asyncio + pytest-xdist for parallel test execution

Quick Start

# Install
pip install hermes-agent

# Run CLI
hermes

# Run as messaging bot
hermes gateway

# Run as MCP server
hermes mcp serve

# Run as ACP server (editor integration)
hermes acp

# Migrate from OpenClaw
hermes claw migrate

DEVBOY Takeaways

  • Self-improving skills = concept ที่ maw ecosystem ควรศึกษา — agent ที่เรียนรู้จาก session ของตัวเอง ไม่ต้องให้คนเขียน skill ใหม่
  • Trajectory compression = training data pipeline ที่ Fleet v3 ยังไม่มี — ถ้า DEVBOY sessions ถูก compress แล้ว fine-tune model ได้ bud จะฉลาดขึ้น
  • MCP server mode = Hermes expose ตัวเองเป็น tool — Claude Desktop browse Hermes conversations ได้ pattern นี้น่าสนใจสำหรับ inter-oracle communication
  • Messaging gateway = 22 platforms จาก codebase เดียว — maw-js plugin system ทำแบบเดียวกันแต่ปัจจุบันมีแค่ Discord
  • Bud signal: ❓ too early — ยังไม่มี application evidence, ต้อง lab test ก่อน
