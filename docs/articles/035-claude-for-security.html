Official Tools — How to Set Up

1. /security-review (Zero setup)

มาพร้อม Claude Code — ใช้ได้เลย:

# ใน Claude Code session:
/security-review

# Claude จะ scan pending changes บน branch ปัจจุบัน
# report: severity, explanation, remediation

2. GitHub Action (PR Security Review)

Add file .github/workflows/claude-security.yml:

name: Claude Security Review
on:
  pull_request:
    types: [opened, synchronize]

jobs:
  security-review:
    runs-on: ubuntu-latest
    permissions:
      contents: read
      pull-requests: write
    steps:
      - uses: actions/checkout@v4
        with:
          fetch-depth: 0
      - uses: anthropics/claude-code-security-review@beta
        with:
          anthropic_api_key: ${{ secrets.ANTHROPIC_API_KEY }}
          model: claude-sonnet-4-6-20250514

Repo: anthropics/claude-code-security-review

Not hardened against prompt injection — ใช้เฉพาะ trusted PRs (internal team). อย่าเปิดรับ PRs จาก untrusted contributors โดยไม่ review.

3. Claude Security (Enterprise)

Enterprise tier → Settings → Security → Enable Claude Security. Features:

  • Scheduled scans — ตั้ง cron scan ทั้ง repo
  • Targeted scans — เลือก specific files/paths
  • Confidence ratings + exploitability explanations
  • Patch generation — auto-fix พร้อม diff
  • Multi-stage false-positive filtering
  • Export to Slack/Jira/CSV

API & Tool Use Patterns

Custom Security Tools (tool_use)

# Define custom tools for Claude to call:
tools = [
    {
        "name": "lookup_cve",
        "description": "Look up CVE details from NVD",
        "input_schema": {
            "type": "object",
            "properties": {
                "cve_id": {"type": "string", "description": "CVE ID e.g. CVE-2024-3094"}
            },
            "required": ["cve_id"]
        }
    },
    {
        "name": "parse_log",
        "description": "Parse and analyze security log entries",
        "input_schema": {
            "type": "object",
            "properties": {
                "log_entry": {"type": "string"},
                "log_type": {"enum": ["cloudtrail", "syslog", "auth", "firewall"]}
            },
            "required": ["log_entry"]
        }
    }
]

# Claude decides when to call these tools
response = client.messages.create(
    model="claude-sonnet-4-6-20250514",
    tools=tools,
    messages=[{"role": "user", "content": "Analyze this CloudTrail event..."}]
)

Limitations

  • Static analysis only — Claude อ่าน code, ไม่ attack running applications
  • Offensive boundaries — ToS ห้ามใช้กับ systems ที่ไม่ได้ own/authorize
  • Mythos gated — model ที่แรงที่สุด restricted ให้ Glasswing partners
  • GitHub Action not injection-hardened — ใช้เฉพาะ trusted PRs
  • Claude Code CVEs — CVE-2025-59536, CVE-2026-21852 (sandbox bypasses, patched)
